Understanding How CSS Works & Priority Rules

1. What is CSS & How It Works?

CSS (Cascading Style Sheets) is the language that tells the browser how to display HTML elements - their colors, sizes, positions, and visual behavior.
Analogy: Think of HTML as the structure of a building (walls, rooms, doors) and CSS as the interior designer who decides the paint colors, furniture arrangement, and lighting. The building exists without the designer, but it looks plain and boring!

<style>
.styled-box {

    text-align: center;
    border: 5px solid #f5576c;
}
</style>
How CSS Works:
1. Browser reads HTML and builds the DOM (Document Object Model)
2. Browser reads CSS and matches selectors to HTML elements
3. Browser applies styles to matching elements
4. Browser renders the styled page on screen
Real-world Analogy: Imagine a Mumbai railway station announcement system:
  • HTML = The announcement board structure (platform numbers, train names)
  • CSS = The display rules (red for delayed trains, green for on-time, font size for platform numbers)
  • Browser = The display system that applies these rules to show colored information

üìù Code Examples:

1. External CSS (styles.css file):

    <link rel="stylesheet" href="styles.css">
</head>

<!-- In styles.css file -->

    background-color: blue;
    color: white;
    padding: 10px;

2. Internal CSS (<style> tag):

<head>
    <style>
        .button {
            background-color: green;
            color: white;
            padding: 10px;
        }
    </style>
</head>

3. Inline CSS (style attribute):

<button style="background-color: red; color: white; padding: 10px;">
    Click Me
</button>
Analogy - Indian Government Hierarchy:

‚Ä¢ Inline CSS = Local Municipal rules (highest priority, applies to specific area)

Just like how local rules override state rules, which override central rules, inline CSS overrides internal CSS, which overrides external CSS!

3. CSS Priority (Cascade) - Live Demo

The Cascade Rule: When multiple CSS rules target the same element, the browser follows a priority system to decide which style to apply. This is called "specificity" and "cascade order."

When to Use Each:

External CSS: Use for entire website styling (colors, fonts, layouts) - Write once, use everywhere!

Internal CSS: Use for page-specific styles that don't apply elsewhere

Inline CSS: Use sparingly for truly unique, one-off styles or dynamic JavaScript changes
