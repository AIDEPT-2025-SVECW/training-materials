1. DataSource (DriverManagerDataSource)

What is DataSource?

DataSource is an interface that provides a standard way to connect to a database. DriverManagerDataSource is Spring's implementation that creates a new connection for each request.

// DataSource Configuration import org.springframework.jdbc.datasource.DriverManagerDataSource; @Configuration public class DatabaseConfig { @Bean public DataSource dataSource() { DriverManagerDataSource dataSource = new DriverManagerDataSource(); dataSource.setDriverClassName("com.mysql.cj.jdbc.Driver"); dataSource.setUrl("jdbc:mysql://localhost:3306/testdb"); dataSource.setUsername("root"); dataSource.setPassword("password"); return dataSource; } }

2. JdbcTemplate

What is JdbcTemplate?

JdbcTemplate is Spring's central class for JDBC operations. It handles resource management, exception handling, and provides a simplified API for database operations.

// JdbcTemplate Configuration import org.springframework.jdbc.core.JdbcTemplate; @Service public class UserService { private JdbcTemplate jdbcTemplate; public UserService(DataSource dataSource) { this.jdbcTemplate = new JdbcTemplate(dataSource); } // Alternative: Auto-inject JdbcTemplate @Autowired private JdbcTemplate jdbcTemplate; }
// Common JdbcTemplate Operations // 1. Query for single value public int getUserCount() { return jdbcTemplate.queryForObject("SELECT COUNT(*) FROM users", Integer.class); } // 2. Query for list of objects public List<User> getAllUsers() { return jdbcTemplate.query("SELECT * FROM users", new UserRowMapper()); } // 3. Update operation public int updateUser(User user) { return jdbcTemplate.update("UPDATE users SET name=?, email=? WHERE id=?", user.getName(), user.getEmail(), user.getId()); } // 4. Insert operation public int insertUser(User user) { return jdbcTemplate.update("INSERT INTO users (name, email) VALUES (?, ?)", user.getName(), user.getEmail()); }

3. RowMapper

What is RowMapper?

RowMapper is a functional interface that maps rows of a ResultSet to Java objects. It's used to convert database rows into POJOs.

// Custom RowMapper Implementation import org.springframework.jdbc.core.RowMapper; import java.sql.ResultSet; import java.sql.SQLException; public class UserRowMapper implements RowMapper<User> { @Override public User mapRow(ResultSet rs, int rowNum) throws SQLException { User user = new User(); user.setId(rs.getLong("id")); user.setName(rs.getString("name")); user.setEmail(rs.getString("email")); user.setCreatedAt(rs.getTimestamp("created_at")); return user; } } // Usage with JdbcTemplate public List<User> findAllUsers() { return jdbcTemplate.query("SELECT * FROM users", new UserRowMapper()); } // Lambda expression alternative public List<User> findAllUsersLambda() { return jdbcTemplate.query("SELECT * FROM users", (rs, rowNum) -> { User user = new User(); user.setId(rs.getLong("id")); user.setName(rs.getString("name")); user.setEmail(rs.getString("email")); return user; }); }

4. BeanPropertyRowMapper

What is BeanPropertyRowMapper?

BeanPropertyRowMapper is a built-in RowMapper that automatically maps database columns to Java bean properties by matching column names to property names.

// BeanPropertyRowMapper Usage import org.springframework.jdbc.core.BeanPropertyRowMapper; // Simple usage - automatic mapping public List<User> findAllUsers() { return jdbcTemplate.query("SELECT * FROM users", new BeanPropertyRowMapper<>(User.class)); } // Alternative static method public List<User> findAllUsersStatic() { return jdbcTemplate.query("SELECT * FROM users", BeanPropertyRowMapper.newInstance(User.class)); } // Single object query public User findUserById(long id) { return jdbcTemplate.queryForObject("SELECT * FROM users WHERE id = ?", new BeanPropertyRowMapper<>(User.class), id); } // User POJO class public class User { private Long id; // maps to 'id' column private String name; // maps to 'name' column private String email; // maps to 'email' column private Date createdAt; // maps to 'created_at' column // Getters and setters required for BeanPropertyRowMapper public Long getId() { return id; } public void setId(Long id) { this.id = id; } // ... other getters and setters }

5. Comparison: Custom RowMapper vs BeanPropertyRowMapper

Custom RowMapper

Pros:

  • Full control over mapping logic
  • Can handle complex transformations
  • Better performance for complex mappings
  • Can handle column name mismatches

Cons:

  • More boilerplate code
  • Need to maintain mapping code
  • Manual error handling
BeanPropertyRowMapper

Pros:

  • Zero configuration for simple mappings
  • Automatic column-to-property mapping
  • Less code to maintain
  • Handles common data types automatically

Cons:

  • Limited control over mapping logic
  • Requires getter/setter methods
  • Column names must match property names
  • Less efficient for simple mappings
