üîç Understanding Transaction Read Problems

When multiple transactions access the database simultaneously, concurrency issues can occur. The most common ones are:

  • Dirty Read: A transaction reads data written by another transaction that has not yet been committed. If the other transaction rolls back, the read data is invalid.
  • Non-Repeatable Read: A row is read twice within a transaction and the data is different because another transaction modified it in between.
  • Phantom Read: A query returns a different number of rows when re-executed within the same transaction because new rows were added by another transaction.

READ_COMMITTED is the default isolation level in most databases (MySQL, PostgreSQL, SQL Server, Oracle)

Think of it as a "Balanced Approach" - Not too strict, not too loose!

READ_UNCOMMITTED

Level: Lowest isolation (Level 0)
