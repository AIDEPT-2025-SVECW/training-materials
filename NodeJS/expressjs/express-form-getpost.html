üöÄ Express.js Form Data Processing

Understanding GET and POST Requests with Practical Examples

üìö Introduction to Form Data Processing

When building web applications, forms are the primary way users send data to your server. Express.js provides elegant ways to handle this data through GET and POST requests.

üéØ Real-World Analogy: Think of your Express server as a restaurant:
  • GET Request: Like ordering from a menu board where everyone can see what you're ordering (data visible in URL)
  • POST Request: Like writing your order on a private note and handing it to the waiter (data hidden in request body)

üîç GET Request - Data in URL

GET requests send data through the URL as query parameters. This is suitable for non-sensitive data like search queries, filters, or pagination.

1. Setting Up the Express Server

const express = require('express');
const app = express();

// Set view engine to EJS
app.set('view engine', 'ejs');

// Serve static files
app.use(express.static('public'));

app.listen(3000, () => {
    console.log('Server running on http://localhost:3000');
});

3. Handling GET Request in Express

// Route to display the search form
app.get('/', (req, res) => {
    res.render('search');
});

// Route to process the search (GET)
app.get('/search', (req, res) => {
    // Access query parameters using req.query
    const studentName = req.query.studentName;
    const city = req.query.city;
    
    console.log('Received GET data:');
    console.log('Student Name:', studentName);
    console.log('City:', city);
    
    // Render results page
    res.render('results', { 
        studentName: studentName, 
        city: city,
        method: 'GET'
    });
});
üîë Key Points about GET:
  • req.query - Contains all query parameters from the URL
  • Data appears in URL: http://localhost:3000/search?studentName=Priya&city=Mumbai
  • Can be bookmarked and shared
  • Limited data size (URL length restrictions)
  • Not secure for sensitive data
üéØ Analogy: GET is like writing your order on the restaurant's whiteboard - everyone can see it, copy it, and refer back to it. Great for "Show me all students from Mumbai" but not for "Here's my password".

üìù POST Request - Data in Request Body

POST requests send data in the request body, making it invisible in the URL. Perfect for sensitive information like passwords, large data, or data that modifies the server state.

1. Middleware Configuration

const express = require('express');
const app = express();

// ESSENTIAL: Middleware to parse POST data
app.use(express.urlencoded({ extended: true })); // For form data
app.use(express.json()); // For JSON data

app.set('view engine', 'ejs');
üîë Critical Middleware:
  • express.urlencoded() - Parses form data (application/x-www-form-urlencoded)
  • extended: true - Allows parsing of rich objects and arrays
  • express.json() - Parses JSON payloads (for APIs)
Without this middleware, req.body will be undefined!

3. Handling POST Request in Express

// Route to display registration form
app.get('/register', (req, res) => {
    res.render('register');
});

// Route to process registration (POST)
app.post('/register', (req, res) => {
    // Access form data using req.body
    const { name, email, password, phone, course, bio } = req.body;
    
    console.log('Received POST data:');
    console.log('Name:', name);
    console.log('Email:', email);
    console.log('Password:', password); // Never log in production!
    console.log('Phone:', phone);
    console.log('Course:', course);
    console.log('Bio:', bio);
    
    // In real application, save to database here
    // For demo, just render success page
    res.render('success', { 
        name: name,
        email: email,
        course: course 
    });
});
üîë Key Points about POST:
  • req.body - Contains all form data from POST request
  • Data is hidden from URL
  • Can send large amounts of data
  • More secure for sensitive information
  • Cannot be bookmarked
  • Used for creating/modifying server data
üéØ Analogy: POST is like sealing your order in an envelope and handing it privately to the waiter. Nobody else sees what you ordered (the URL stays clean), and you can send a detailed, lengthy order without worrying about space.

‚öñÔ∏è GET vs POST - When to Use What?

‚úÖ Use GET When:

  • Searching/filtering data
  • Data is not sensitive
  • User might want to bookmark
  • Pagination (page numbers)
  • Retrieving data only
  • Example: Search for students from "Bangalore"

‚úÖ Use POST When:

  • Submitting sensitive data (passwords)
  • Creating new records
  • Uploading files
  • Large amounts of data
  • Modifying server state
  • Example: User registration form
‚ö†Ô∏è Common Mistakes:
  • Forgetting express.urlencoded() middleware for POST requests
  • Using GET for passwords or sensitive data
  • Not matching form action URL with Express route
  • Mismatching form method (GET/POST) with Express route method

// server.js - Complete Express Application
const express = require('express');
const app = express();
const PORT = 3000;

// Middleware
app.use(express.urlencoded({ extended: true }));
app.use(express.json());
app.use(express.static('public'));

// Set EJS as templating engine
app.set('view engine', 'ejs');

// In-memory storage (use database in production)
let students = [
    { name: 'Priya Sharma', email: 'priya@example.com', course: 'Computer Science', city: 'Mumbai' },
    { name: 'Arjun Patel', email: 'arjun@example.com', course: 'Electronics', city: 'Delhi' },
    { name: 'Meera Reddy', email: 'meera@example.com', course: 'Mechanical', city: 'Bangalore' }
];

// HOME PAGE
app.get('/', (req, res) => {
    res.render('home');
});

// GET REQUEST DEMO - Search
app.get('/search', (req, res) => {
    res.render('search', { results: null });
});

app.get('/search-results', (req, res) => {
    const { studentName, city } = req.query;
    
    // Filter students based on search criteria
    const results = students.filter(student => {
        const nameMatch = studentName ? 
            student.name.toLowerCase().includes(studentName.toLowerCase()) : true;
        const cityMatch = city ? student.city === city : true;
        return nameMatch && cityMatch;
    });
    
    res.render('search', { 
        results: results, 
        searchTerm: studentName, 
        searchCity: city 
    });
});

// POST REQUEST DEMO - Registration
app.get('/register', (req, res) => {
    res.render('register');
});

app.post('/register', (req, res) => {
    const { name, email, password, phone, course, bio } = req.body;
    
    // Validate data (basic example)
    if (!name || !email || !password) {
        return res.status(400).send('Missing required fields');
    }
    
    // Add to students array (in production, save to database)
    students.push({
        name: name,
        email: email,
        course: course,
        city: 'New Registration'
    });
    
    console.log('New student registered:', name);
    console.log('Total students:', students.length);
    
    res.render('success', { 
        name: name,
        email: email,
        course: course 
    });
});

// View all students
app.get('/students', (req, res) => {
    res.render('students', { students: students });
});

// Start server
app.listen(PORT, () => {
    console.log(`Server is running on http://localhost:${PORT}`);
    console.log('Available routes:');
    console.log('  - GET  /');
    console.log('  - GET  /search');
    console.log('  - GET  /search-results?studentName=...&city=...');
    console.log('  - GET  /register');
    console.log('  - POST /register');
    console.log('  - GET  /students');
});

1. Accessing Different Types of Data

// URL Parameters (route parameters)
app.get('/student/:id', (req, res) => {
    const studentId = req.params.id;
    // Access: /student/12345
    res.send(`Student ID: ${studentId}`);
});

// Query Parameters (GET)
app.get('/filter', (req, res) => {
    const { course, year, city } = req.query;
    // Access: /filter?course=CS&year=2024&city=Mumbai

});

app.post('/submit', (req, res) => {
    const { name, email, age } = req.body;

2. Data Validation and Sanitization

app.post('/register', (req, res) => {
    const { name, email, age } = req.body;
    
    // Basic validation
    if (!name || name.trim() === '') {
        return res.status(400).send('Name is required');
    }
    
    if (!email || !email.includes('@')) {
        return res.status(400).send('Valid email is required');
    }
    
    if (age && (isNaN(age) || age < 18)) {
        return res.status(400).send('Age must be 18 or above');
    }
    
    // Sanitize data
    const sanitizedName = name.trim();
    const sanitizedEmail = email.toLowerCase().trim();
    
    // Process the valid data
    res.send('Registration successful!');
});

4. Redirecting After POST (Post/Redirect/Get Pattern)

app.post('/register', (req, res) => {
    const { name, email } = req.body;
    
    // Save to database
    // ... database logic ...
    
    // Redirect to success page (GET request)
    // This prevents form resubmission on page refresh
    res.redirect('/success?name=' + encodeURIComponent(name));
});

app.get('/success', (req, res) => {
    const name = req.query.name;
    res.render('success', { name: name });
});
üîë Best Practice - Post/Redirect/Get (PRG) Pattern: After processing a POST request, always redirect to a GET page. This prevents the "Do you want to resubmit the form?" browser warning when users refresh the page.

üîß Common Issues & Solutions

Issue 1: req.body is undefined

Problem: POST data is not accessible 
// ‚ùå WRONG - Missing middleware
const express = require('express');
const app = express();

app.post('/submit', (req, res) => {
    console.log(req.body); // undefined!
});

// ‚úÖ CORRECT - Add middleware BEFORE routes
const express = require('express');
const app = express();

app.use(express.urlencoded({ extended: true })); // Add this!
app.use(express.json()); // For JSON data

app.post('/submit', (req, res) => {
    console.log(req.body); // Now it works!
});

Issue 2: Form action doesn't match route

<!-- ‚ùå WRONG -->
<form action="/register" method="POST">
    <!-- form fields -->
</form>

// Express route doesn't match
app.post('/registration', (req, res) => { // Different path!
    // This won't be called
});

<!-- ‚úÖ CORRECT -->
<form action="/register" method="POST">
    <!-- form fields -->
</form>

app.post('/register', (req, res) => { // Matching path
    // This will work
});

Issue 3: Method mismatch

<!-- ‚ùå WRONG - Form uses GET but route expects POST -->
<form action="/submit" method="GET">
    <!-- form fields -->
</form>

app.post('/submit', (req, res) => {
    // This won't be called - method mismatch!
});

<!-- ‚úÖ CORRECT - Methods match -->
<form action="/submit" method="POST">
    <!-- form fields -->
</form>

app.post('/submit', (req, res) => {
    // This works!
});

üìã Quick Reference Summary

GET Request Checklist:
  • ‚úÖ Form method: <form method="GET">
  • ‚úÖ Express route: app.get('/path', ...)
  • ‚úÖ Access data: req.query.fieldName
  • ‚úÖ Data visible in URL
  • ‚úÖ No middleware needed
  • ‚úÖ Use for: Search, filter, pagination
POST Request Checklist:
  • ‚úÖ Form method: <form method="POST">
  • ‚úÖ Express route: app.post('/path', ...)
  • ‚úÖ Access data: req.body.fieldName
  • ‚úÖ Middleware: app.use(express.urlencoded({ extended: true }))
  • ‚úÖ Data hidden from URL
  • ‚úÖ Use for: Forms, sensitive data, creating records
üéØ Final Analogy: Think of your Express server as a receptionist at a company:
  • GET: Someone walks up and asks loudly, "Where is the conference room?" (Everyone hears - public information)
  • POST: Someone hands the receptionist a sealed envelope with confidential documents (Private - only the receptionist sees it)
Both get the job done, but you choose based on the sensitivity and nature of the information!
