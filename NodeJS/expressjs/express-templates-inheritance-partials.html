ğŸ“š What are Template Engines?

Template engines in Express allow you to use static template files in your application. At runtime, the template engine replaces variables in a template file with actual values and transforms the template into an HTML file sent to the client.

Real-World Analogy

Think of templates like a restaurant menu template. Every restaurant has a similar menu structure (header, sections, footer), but the actual dishes and prices are different. Instead of designing a new menu from scratch each time, you use a template and just fill in the specific details!

ğŸ—ï¸ Template Inheritance (Layouts)

Template inheritance allows you to create a base "skeleton" template that contains all the common elements of your site (header, footer, navigation) and then extend it in other templates.

Building Construction Analogy

Imagine building apartments in a building. The base layout is like the building's structure (foundation, walls, plumbing, electricity). Each apartment (page) inherits this structure but has its own interior design (content). You don't rebuild the entire structure for each apartment!

ğŸ§© Partials (Reusable Components)

Partials are small, reusable template fragments that can be included in multiple templates. They're like LEGO blocks - small pieces you can use anywhere to build bigger things.

LEGO Blocks Analogy

Think of partials as LEGO blocks. You have standard pieces (header, footer, card, button) that you can snap into different creations (pages). Instead of crafting each piece every time, you reuse the same blocks across multiple builds!

Including Partials in Templates

<!-- views/menu.ejs -->
<!DOCTYPE html>
<html>
<head>
    <title>Menu - Priya's Restaurant</title>
</head>
<body>
    <!-- Include header partial -->
    <%- include('partials/header') %>

    <main>
        <h1>Our Menu</h1>
        
        <div class="menu-grid">
            <% dishes.forEach(dish => { %>
                <!-- Include dish-card partial for each dish -->
                <%- include('partials/dish-card', {dish: dish}) %>
            <% }); %>
        </div>
    </main>

    <!-- Include footer partial -->
    <%- include('partials/footer') %>
</body>
</html>

ğŸ”‘ Key Benefits of Partials:

  • Reusability: Write once, use everywhere (headers, footers, cards)
  • Consistency: UI components look the same across pages
  • Maintainability: Update one partial, changes reflect everywhere
  • Modularity: Break complex pages into manageable pieces
  • Dynamic Content: Pass different data to same partial

âš–ï¸ Layout vs Partials: When to Use What?

ğŸ—ï¸ Use Layouts When:

  • You need a consistent page structure
  • Header and footer are same across pages
  • You want to define overall page skeleton
  • Managing SEO meta tags centrally

Example: All pages need same header, footer, and meta tags

ğŸ§© Use Partials When:

  • You have reusable UI components
  • Same element appears multiple times
  • Need to pass different data to same template
  • Breaking down complex pages

Example: Product cards, user cards, alert messages, forms

Main Application Setup

// app.js
const express = require('express');
const expressLayouts = require('express-ejs-layouts');
const path = require('path');

const app = express();

// View engine setup
app.set('view engine', 'ejs');
app.set('views', path.join(__dirname, 'views'));

// Use express-ejs-layouts
app.use(expressLayouts);
app.set('layout', 'layouts/main');

// Static files
app.use(express.static('public'));

// Body parser
app.use(express.urlencoded({ extended: true }));

// Routes
const indexRouter = require('./routes/index');
app.use('/', indexRouter);

// Start server
const PORT = process.env.PORT || 3000;
app.listen(PORT, () => {
    console.log(`Server running on http://localhost:${PORT}`);
});

âœ¨ Best Practices & Tips

ğŸ“‹ Naming Conventions:

  • Use descriptive names: user-card.ejs not uc.ejs
  • Keep partials in a dedicated partials/ folder
  • Layouts in layouts/ folder for organization

ğŸ¯ When to Create a Partial:

  • Component used in multiple places
  • Component has complex structure worth isolating
  • You want to reuse with different data
  • Makes the main template cleaner and more readable

âš ï¸ Common Mistakes to Avoid:

  • Don't create too many tiny partials - balance is key
  • Avoid deeply nested partials (partials including partials including partials)
  • Remember: <%- include() %> renders HTML, <%= %> escapes it
  • Always pass required data to partials to avoid undefined errors
