üß¨ What are Prototypes?

üèóÔ∏è Blueprint Analogy

Think of prototypes like architectural blueprints. Just as a house blueprint contains the design that many houses can follow, a JavaScript prototype contains properties and methods that many objects can inherit. Each house (object) built from the blueprint has access to all the rooms and features defined in the blueprint, but can also have its own unique additions.

In JavaScript, every object has a hidden link to another object called its prototype. When you try to access a property that doesn't exist on an object, JavaScript automatically looks up the prototype chain to find it.

Basic Prototype Example:

// Every function has a prototype property
function Person(name, age) {
    this.name = name;
    this.age = age;
}

// Adding methods to the prototype
Person.prototype.greet = function() {
    return `Hello, I'm ${this.name} and I'm ${this.age} years old.`;
};

Person.prototype.isAdult = function() {
    return this.age >= 18;
};

// Creating instances
const person1 = new Person('Alice', 25);
const person2 = new Person('Bob', 17);

// Both objects can use prototype methods
console.log(person1.greet()); // "Hello, I'm Alice and I'm 25 years old."
console.log(person2.isAdult()); // false

üîó The Prototype Chain

üéØ Delegation Chain Analogy

Imagine a company hierarchy where if an employee doesn't know how to do something, they ask their manager. If the manager doesn't know, they ask their director, and so on up the chain. JavaScript's prototype chain works similarly - if an object doesn't have a property, it asks its prototype, then the prototype's prototype, until it reaches the top (null).

// Demonstrating the prototype chain
function Animal(species) {
    this.species = species;
}

Animal.prototype.breathe = function() {
    return `${this.species} is breathing`;
};

function Dog(name, breed) {
    Animal.call(this, 'Canine'); // Call parent constructor
    this.name = name;
    this.breed = breed;
}

// Set up inheritance
Dog.prototype = Object.create(Animal.prototype);
Dog.prototype.constructor = Dog;

// Add Dog-specific methods
Dog.prototype.bark = function() {
    return `${this.name} says Woof!`;
};

const myDog = new Dog('Buddy', 'Golden Retriever');

// myDog can access methods from both Dog and Animal prototypes
console.log(myDog.bark());    // "Buddy says Woof!" (from Dog.prototype)
console.log(myDog.breathe()); // "Canine is breathing" (from Animal.prototype)

üé≠ Modern Inheritance with Classes

üè´ School Class Analogy

ES6 classes are like creating a curriculum for a school subject. The base class (like "Science") defines general concepts, while specialized classes (like "Biology" or "Chemistry") extend the base class with more specific knowledge. Students (instances) learn both the general and specialized concepts.

// Modern ES6 class syntax
class Animal {
    constructor(species, name) {
        this.species = species;
        this.name = name;
    }
    
    breathe() {
        return `${this.name} the ${this.species} is breathing`;
    }
    
    move() {
        return `${this.name} is moving`;
    }
}

class Bird extends Animal {
    constructor(name, canFly = true) {
        super('Bird', name); // Call parent constructor
        this.canFly = canFly;
    }
    
    // Override parent method
    move() {
        if (this.canFly) {
            return `${this.name} is flying`;
        } else {
            return super.move(); // Call parent method
        }
    }
    
    chirp() {
        return `${this.name} says chirp chirp!`;
    }
}

class Penguin extends Bird {
    constructor(name) {
        super(name, false); // Penguins can't fly
    }
    
    swim() {
        return `${this.name} is swimming gracefully`;
    }
    
    // Override chirp method
    chirp() {
        return `${this.name} says squawk squawk!`;
    }
}

üîç Key Concepts Summary

1. Prototype Property vs __proto__

// Functions have a 'prototype' property
function MyFunction() {}
console.log(MyFunction.prototype); // Object with constructor property

// Objects have a __proto__ property (internal [[Prototype]])
const obj = new MyFunction();
console.log(obj.__proto__ === MyFunction.prototype); // true

2. Constructor Functions vs Classes

// Constructor function approach (older)
function Car(make, model) {
    this.make = make;
    this.model = model;
}
Car.prototype.start = function() { /* ... */ };

// Class approach (ES6+, preferred)
class Car {
    constructor(make, model) {
        this.make = make;
        this.model = model;
    }
    start() { /* ... */ }
}

3. Important Methods

// Object.create() - creates object with specific prototype
const parent = { x: 1, y: 2 };
const child = Object.create(parent);

// Object.getPrototypeOf() - gets object's prototype
console.log(Object.getPrototypeOf(child) === parent); // true

// hasOwnProperty() - checks if property is own (not inherited)
console.log(child.hasOwnProperty('x')); // false (inherited)
child.z = 3;
console.log(child.hasOwnProperty('z')); // true (own property)
