Understanding package management in Node.js

üéØ What is npm?

npm (Node Package Manager) is the default package manager for Node.js. It helps you install, update, and manage JavaScript packages (libraries/tools) in your projects.

üìö Analogy: Think of npm as a library system. Local installation is like keeping books in your personal study room for a specific project, while global installation is like keeping books in a central library that all your study rooms can access.

üè† Local Installation

Installs packages in the node_modules folder of your current project directory.

Command:

npm install package-name
# or shorthand
npm i package-name

# Save as development dependency
npm install package-name --save-dev
# or shorthand
npm i package-name -D

Example:

# Install lodash locally
npm install lodash

# Install Jest as dev dependency
npm install jest --save-dev

Where it goes:

my-project/

‚úÖ When to use Local Installation:

  • Project dependencies - Libraries your code imports and uses (lodash, express, react)
  • Development tools - Testing frameworks, build tools specific to your project (jest, webpack)
  • Different versions - When different projects need different versions of the same package
  • Team collaboration - Ensures everyone on the team uses the same package versions

How to use in code:

// Import and use the locally installed package
const lodash = require('lodash');

const array = [1, 2, 3, 4, 5];
const doubled = lodash.map(array, n => n * 2);
console.log(doubled); // [2, 4, 6, 8, 10]

üåç Global Installation

Installs packages in a system-wide location, making them available from anywhere on your computer.

Command:

npm install -g package-name
# or
npm install --global package-name

# Example
npm install -g nodemon

Where it goes:

# On Unix/Mac:
/usr/local/lib/node_modules/

# On Windows:
C:\Users\YourName\AppData\Roaming\npm\node_modules\

# Check global location:
npm root -g

‚úÖ When to use Global Installation:

  • Command-line tools - Utilities you run from terminal (nodemon, create-react-app, npm)
  • Development utilities - Tools used across multiple projects (http-server, eslint)
  • System-wide access - When you need to run commands from any directory

Example with nodemon:

# Install globally
npm install -g nodemon

# Now you can use it from any project directory
nodemon app.js

# Without global install, you'd get:
# nodemon: command not found
‚ö†Ô∏è Warning: Global installations can cause version conflicts between projects. Modern best practice is to use local installations with npx or npm scripts whenever possible.

üìÅ Local Installation

Command: npm install package-name

Location: ./node_modules/

Scope: Current project only

Listed in: package.json

Best for:

  • Project dependencies
  • Libraries you import
  • Version-specific needs

Examples:

  • express
  • react
  • lodash
  • jest

üåê Global Installation

Command: npm install -g package-name

Location: System directory

Scope: Entire system

Listed in: Not in project's package.json

Best for:

  • CLI tools
  • Development utilities
  • System-wide commands

Examples:

  • nodemon
  • http-server
  • create-react-app
  • typescript

üîç Quick Reference Commands

# View all globally installed packages
npm list -g --depth=0

# View locally installed packages
npm list --depth=0

# Uninstall local package
npm uninstall package-name

# Uninstall global package
npm uninstall -g package-name

# Check where global packages are installed
npm root -g

# Update global package
npm update -g package-name

# Update all global packages
npm update -g

üí° Key Takeaways

  • Default to local installations - Use npm install package-name for most packages
  • Global only for CLI tools - Install globally only when you need command-line access everywhere
  • Use npx when possible - Avoid global installs by using npx for one-off commands
  • Check package.json - Local installations are tracked here, making your project reproducible
  • Team consistency - Local installations ensure everyone uses the same versions
