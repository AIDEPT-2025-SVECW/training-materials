üè† Real-World Analogy

Think of the File System as your house's filing cabinet:

  • Files are like documents in folders
  • Directories are like drawer compartments
  • Reading files is like opening a document to read it
  • Writing files is like writing on a document and saving it
  • Creating directories is like adding new compartments to your filing cabinet

üìÅ Basic File Operations

üìñ Reading Files - fs.readFile()

Like opening a book to read its contents. The computer opens the file and brings all its content into memory.

const fs = require('fs');

// Asynchronous read
fs.readFile('example.txt', 'utf8', (err, data) => {
    if (err) {
        console.error('Error reading file:', err);
        return;
    }
    console.log('File content:', data);
});

// Synchronous read
try {
    const data = fs.readFileSync('example.txt', 'utf8');
    console.log('File content:', data);
} catch (err) {
    console.error('Error reading file:', err);
}
‚úçÔ∏è Writing Files - fs.writeFile()

Like writing on a blank paper or completely replacing the content of an existing document.

const fs = require('fs');

// Write to file (overwrites existing content)
const content = 'Hello, World! This is new content.';

fs.writeFile('example.txt', content, 'utf8', (err) => {
    if (err) {
        console.error('Error writing file:', err);
        return;
    }
    console.log('File written successfully!');
});

// Synchronous write
try {
    fs.writeFileSync('example.txt', content, 'utf8');
    console.log('File written successfully!');
} catch (err) {
    console.error('Error writing file:', err);
}
‚ûï Appending to Files - fs.appendFile()

Like adding new pages to the end of an existing book without removing the old content.

const fs = require('fs');

// Append to file
const additionalContent = '\nThis is appended content.';

fs.appendFile('example.txt', additionalContent, 'utf8', (err) => {
    if (err) {
        console.error('Error appending to file:', err);
        return;
    }
    console.log('Content appended successfully!');
});

‚ö° Asynchronous vs Synchronous Operations

üçΩÔ∏è Restaurant Analogy

Synchronous (Sync): Like a single-person kitchen where the chef must complete one order entirely before starting the next. Everything happens in sequence.

Asynchronous (Async): Like a restaurant kitchen with multiple chefs working on different orders simultaneously. Tasks can be started and completed in different orders.

‚è≥ Synchronous Operations

Blocking operations that wait for completion before moving to the next line.

const fs = require('fs');

console.log('Start');

// This blocks until file is read
try {
    const data = fs.readFileSync('file1.txt', 'utf8');
    console.log('File 1:', data);
} catch (err) {
    console.error('Error:', err.message);
}

// This only runs AFTER the above completes
try {
    const data2 = fs.readFileSync('file2.txt', 'utf8');
    console.log('File 2:', data2);
} catch (err) {
    console.error('Error:', err.message);
}

console.log('End');
üöÄ Asynchronous Operations

Non-blocking operations that allow other code to run while waiting for file operations to complete.

const fs = require('fs');

console.log('Start');

// These operations start immediately but complete later
fs.readFile('file1.txt', 'utf8', (err, data) => {
    if (err) {
        console.error('Error reading file1:', err.message);
    } else {
        console.log('File 1:', data);
    }
});

fs.readFile('file2.txt', 'utf8', (err, data) => {
    if (err) {
        console.error('Error reading file2:', err.message);
    } else {
        console.log('File 2:', data);
    }
});

console.log('End'); // This runs immediately!

üåä File Streams

üö∞ Water Pipe Analogy

Streams are like water pipes: Instead of filling a giant bucket with all the water at once (loading entire file), streams let water flow through pipes piece by piece. This is much more memory-efficient for large files!

üì• Reading Streams

Process large files piece by piece without loading everything into memory at once.

const fs = require('fs');

// Create a readable stream
const readStream = fs.createReadStream('large-file.txt', {
    encoding: 'utf8',
    highWaterMark: 16 * 1024 // 16KB chunks
});

readStream.on('data', (chunk) => {
    console.log('Received chunk:', chunk.length, 'characters');
    // Process chunk here
});

readStream.on('end', () => {
    console.log('Finished reading file');
});

readStream.on('error', (err) => {
    console.error('Error reading stream:', err);
});
üì§ Writing Streams

Write data to files piece by piece, useful for generating large files or continuous data.

const fs = require('fs');

// Create a writable stream
const writeStream = fs.createWriteStream('output.txt');

// Write data in chunks
writeStream.write('First chunk of data\n');
writeStream.write('Second chunk of data\n');
writeStream.write('Third chunk of data\n');

// End the stream
writeStream.end();

writeStream.on('finish', () => {
    console.log('Writing finished');
});

writeStream.on('error', (err) => {
    console.error('Error writing stream:', err);
});

üîß Advanced File System Operations

üìä File Statistics - fs.stat()

Get detailed information about files and directories, like checking a file's ID card.

const fs = require('fs');

fs.stat('example.txt', (err, stats) => {
    if (err) {
        console.error('Error getting file stats:', err);
        return;
    }
    
    console.log('File Statistics:');
    console.log('Size:', stats.size, 'bytes');
    console.log('Is File:', stats.isFile());
    console.log('Is Directory:', stats.isDirectory());
    console.log('Created:', stats.birthtime);
    console.log('Modified:', stats.mtime);
    console.log('Accessed:', stats.atime);
});
üìÅ Directory Operations

Create, read, and manage directories - like organizing your filing cabinet.

const fs = require('fs');
const path = require('path');

// Create directory
fs.mkdir('new-folder', { recursive: true }, (err) => {
    if (err) {
        console.error('Error creating directory:', err);
    } else {
        console.log('Directory created successfully');
    }
});

// Read directory contents
fs.readdir('.', (err, files) => {
    if (err) {
        console.error('Error reading directory:', err);
    } else {
        console.log('Directory contents:', files);
    }
});

// Remove directory
fs.rmdir('old-folder', (err) => {
    if (err) {
        console.error('Error removing directory:', err);
    } else {
        console.log('Directory removed successfully');
    }
});
üîç File Watching - fs.watch()

Monitor files and directories for changes, like having a security guard watch your files.

const fs = require('fs');

// Watch for file changes
const watcher = fs.watch('example.txt', (eventType, filename) => {
    if (filename) {
        console.log(\`File \${filename} was \${eventType}d\`);
    }
});

// Watch for directory changes
const dirWatcher = fs.watch('.', (eventType, filename) => {
    if (filename) {
        console.log(\`Directory change: \${filename} was \${eventType}d\`);
    }
});

// Stop watching after 30 seconds
setTimeout(() => {
    watcher.close();
    dirWatcher.close();
    console.log('Stopped watching files');
}, 30000);
