üìö What Are Static Files?

Static files are files that don't change when served to different users. They're delivered exactly as stored on the server:

  • HTML files - Your web pages
  • CSS files - Styling and design
  • JavaScript files - Client-side logic
  • Images (PNG, JPG, SVG) - Visual content
  • Fonts, Videos, PDFs - Other media

üéØ Real-World Analogy: The Library

Think of Node.js as a librarian and your server as a library. When someone (a browser) asks for a book (a file), the librarian finds it on the shelf (file system) and hands it over unchanged. The librarian doesn't rewrite the book for each reader‚Äîeveryone gets the same content!

Method 1: Basic HTTP Server

The most fundamental way‚Äîmanually reading and serving files.

const http = require('http');
const fs = require('fs');
const path = require('path');

const server = http.createServer((req, res) => {
    // Construct file path
    let filePath = path.join(__dirname, 'public', req.url === '/' ? 'index.html' : req.url);
    
    // Get file extension
    const extname = path.extname(filePath);
    
    // Set content type based on file extension
    let contentType = 'text/html';
    switch (extname) {
        case '.css': contentType = 'text/css'; break;
        case '.js': contentType = 'text/javascript'; break;
        case '.json': contentType = 'application/json'; break;
        case '.png': contentType = 'image/png'; break;
        case '.jpg': contentType = 'image/jpg'; break;
    }
    
    // Read and serve the file
    fs.readFile(filePath, (err, content) => {
        if (err) {
            if (err.code === 'ENOENT') {
                // File not found
                res.writeHead(404, { 'Content-Type': 'text/html' });
                res.end('<h1>404 - File Not Found</h1>');
            }
        } else {
            // Success!
            res.writeHead(200, { 'Content-Type': contentType });
            res.end(content);
        }
    });
});

server.listen(3000, () => console.log('Server running on port 3000'));
Key Concepts:
  • Content-Type header: Tells the browser what type of file it's receiving
  • fs.readFile(): Reads the entire file into memory
  • Error handling: Important for 404 errors

Method 2: Using fs.createReadStream (Better Performance)

For larger files, streaming is more efficient than reading the entire file at once.

const http = require('http');
const fs = require('fs');
const path = require('path');

const server = http.createServer((req, res) => {
    const filePath = path.join(__dirname, 'public', req.url === '/' ? 'index.html' : req.url);
    
    // Check if file exists
    fs.access(filePath, fs.constants.F_OK, (err) => {
        if (err) {
            res.writeHead(404);
            res.end('File not found');
            return;
        }
        
        // Create a read stream - sends data in chunks
        const readStream = fs.createReadStream(filePath);
        
        res.writeHead(200);
        
        // Pipe the file directly to the response
        readStream.pipe(res);
        
        // Handle stream errors
        readStream.on('error', (error) => {
            res.writeHead(500);
            res.end('Server error');
        });
    });
});

server.listen(3000);

üíß Streaming Analogy: The Water Pipe

Instead of filling a huge bucket (memory) with all the water (file data) before pouring it out, streaming is like connecting a pipe directly from the source to the destination. Water flows continuously in small amounts‚Äîmuch more efficient for large volumes!

Method 3: Express.js (Recommended for Production)

Express provides a simple, one-line solution with built-in optimizations.

// First install: npm install express

const express = require('express');
const path = require('path');

const app = express();

// Magic line! Serves all files in 'public' directory
app.use(express.static(path.join(__dirname, 'public')));

// Optional: Set cache control headers
app.use(express.static('public', {
    maxAge: '1d',  // Cache for 1 day
    etag: true,     // Enable ETag for caching
    index: 'index.html'  // Default file
}));

// Fallback route for SPA (Single Page Applications)
app.get('*', (req, res) => {
    res.sendFile(path.join(__dirname, 'public', 'index.html'));
});

app.listen(3000, () => {
    console.log('Express server running on http://localhost:3000');
});
Why Express is Better:
  • ‚úÖ Automatic content-type detection
  • ‚úÖ Built-in error handling
  • ‚úÖ Caching headers management
  • ‚úÖ Range requests for video streaming
  • ‚úÖ ETag support for efficient caching
  • ‚úÖ Security features like path traversal prevention

‚ö° Best Practices & Tips

1. Always Use Path Module

// ‚ùå BAD - Can cause issues on different operating systems
const filePath = __dirname + '/public/' + req.url;

// ‚úÖ GOOD - Cross-platform compatible
const filePath = path.join(__dirname, 'public', req.url);

2. Set Proper MIME Types

MIME types tell the browser how to handle the file:

const mimeTypes = {
    '.html': 'text/html',
    '.css': 'text/css',
    '.js': 'text/javascript',
    '.json': 'application/json',
    '.png': 'image/png',
    '.jpg': 'image/jpeg',
    '.gif': 'image/gif',
    '.svg': 'image/svg+xml',
    '.ico': 'image/x-icon',
    '.pdf': 'application/pdf'
};

3. Implement Caching

// Tell browsers to cache static files for 1 week
res.setHeader('Cache-Control', 'public, max-age=604800');

4. Security: Prevent Directory Traversal

// ‚ùå DANGEROUS - Could access files outside public directory
// URL: ../../../etc/passwd

// ‚úÖ SAFE - Validate paths
const safePath = path.normalize(filePath).replace(/../g, '');
if (!safePath.startsWith(__dirname)) {
    res.writeHead(403);
    res.end('Forbidden');
}
