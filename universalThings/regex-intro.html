What is a Regular Expression?
A special pattern that searches, matches, and validates text — one of the most powerful tools in a developer's toolkit.
Simple Analogy: Think of a regex like a stencil. Just as a stencil lets you draw a specific shape repeatedly, a regex lets you find and match specific text patterns — as many times as they appear in a document.
🔍Anatomy of a RegEx

Every regex expression has three parts — here is a real example for an Indian mobile number:

/Start
[6-9]\d{9}Pattern
/End
giFlags
  • 1
    Forward Slashes ( / / ) — The boundary of a regex. In JavaScript, the pattern is written between these two slashes.
  • 2
    Pattern — The actual matching rules are written here. This is the core of what we will learn.
  • 3
    Flags — Optional modifiers. g = find all matches (global), i = ignore uppercase/lowercase, m = multiline mode.

🚦How Does RegEx Work? — Step by Step
  • 1
    You write a pattern that describes the text structure you are looking for (for example, the format of a 10-digit mobile number).
  • 2
    The regex engine scans the input text from left to right, testing your pattern at every position.
  • 3
    Whenever the pattern matches a portion of the text, the engine records that position and the matched value.
  • 4
    You can then use the result to validate, extract, or replace text inside your program.
Regex Syntax Guide

🔤Character Classes — "What Characters to Match?"
Symbol Meaning Example Matches
. Any single character (except newline) a.c abc, a1c, a@c
\d Any digit — same as [0-9] \d\d 12, 45, 99
\D Any NON-digit character \D+ abc, @#$, hello
\w Word character (a-z, A-Z, 0-9, _) \w+ hello, user_1, Ram
\W Non-word character (spaces, symbols) \W space, @, !
\s Whitespace (space, tab, newline) hello\sworld "hello world"
[abc] Either a, b, or c — a set of options [aeiou] a, e, i, o, u
[^abc] Anything except a, b, or c [^0-9] letters, symbols
[a-z] Any character in a range [A-Z] A, B, C … Z
🔢Quantifiers — "How Many Times?"
Symbol Meaning Example Matches
* Zero or more times go*d gd, god, good, goood
+ One or more times go+d god, good — but NOT gd
? Zero or one time (makes it optional) colou?r color, colour
{n} Exactly n times \d{4} 1234, 9876
{n,} n or more times \d{6,} 123456, 12345678
{n,m} Between n and m times (inclusive) \d{6,10} Any 6 to 10 digit number
⚓Anchors & Boundaries — "Where to Match?"
Symbol Meaning Example Effect
^ Start of the string ^\d{10} Must begin with 10 digits
$ End of the string \d{10}$ Must end with 10 digits
^…$ Full string match (start to end) ^\d{10}$ Entire string must be 10 digits
\b Word boundary \bcat\b Matches "cat" but not "catch"
| OR — matches either side cat|dog cat or dog
() Group — treat as a single unit (ab)+ ab, abab, ababab

Ready-made regex patterns for real Indian documents and data formats.

📱 Indian Mobile Number
10-digit number starting with 6, 7, 8, or 9 — the only valid Indian mobile prefixes.
/^[6-9]\d{9}$/
✅ Valid: 9876543210
❌ Invalid: 1234567890 (starts with 1)
🪪 Aadhaar Number
12-digit unique ID issued by UIDAI. Spaces between groups are optional.
/^\d{4}\s?\d{4}\s?\d{4}$/
✅ Valid: 1234 5678 9012
❌ Invalid: 12345678 (only 8 digits)
📋 PAN Card Number
5 uppercase letters + 4 digits + 1 uppercase letter. Format: AAAAA0000A
/^[A-Z]{5}[0-9]{4}[A-Z]{1}$/
✅ Valid: ABCDE1234F
❌ Invalid: ABC123456F (only 3 letters at start)
📮 Indian PIN Code
6-digit postal code. First digit must be 1–9 (PIN codes never start with 0).
/^[1-9][0-9]{5}$/
✅ Valid: 110001 (New Delhi)
❌ Invalid: 012345 (starts with 0)
📧 Email Address
Standard email format. Works for Gmail, Yahoo, and Indian .co.in domains.
/^[\w.-]+@[\w.-]+\.\w{2,}$/
✅ Valid: ram.sharma@gmail.com
❌ Invalid: ram@.com (dot right after @)
🚗 Vehicle Registration Number
2-letter state code + 2-digit RTO code + 1–2 letter series + 4-digit number.
/^[A-Z]{2}[0-9]{2}[A-Z]{1,2}[0-9]{4}$/
✅ Valid: MH01AB1234
❌ Invalid: MHAB1234 (missing RTO digits)
🏦 IFSC Code
4-letter bank code + literal 0 + 6 alphanumeric branch characters.
/^[A-Z]{4}0[A-Z0-9]{6}$/
✅ Valid: SBIN0001234 (SBI branch)
❌ Invalid: SBI001234 (too short)
📅 Indian Date Format
DD/MM/YYYY — the standard date format used across India.
/^(0[1-9]|[12]\d|3[01])\/(0[1-9]|1[0-2])\/\d{4}$/
✅ Valid: 15/08/1947 (Independence Day)
❌ Invalid: 1947-08-15 (wrong format)
💰 Indian Rupee Amount
Indian number format with lakhs and crores (e.g., ₹1,50,000 not ₹150,000).
/^₹?\d{1,2}(,\d{2})*,\d{3}(\.\d{2})?$/
✅ Valid: ₹1,50,000 (1.5 lakh)
✅ Valid: ₹45,00,000 (45 lakh)
🌐 Indian Domain (.in / .co.in)
Matches Indian TLD domains including government and educational domains.
/^[\w-]+\.(in|co\.in|gov\.in|edu\.in)$/
✅ Valid: mygov.in, cbse.gov.in
❌ Invalid: google.com (not .in)
📊 GST Number (GSTIN)
15-character Goods and Services Tax Identification Number format.
/^\d{2}[A-Z]{5}\d{4}[A-Z][1-9A-Z]Z[0-9A-Z]$/
✅ Valid: 27AAAPL1234C1ZV
❌ Invalid: 27AAPL1234C1ZV (only 4 letters)
🔑 Strong Password
Minimum 8 characters with at least 1 uppercase letter, 1 digit, and 1 special character.
/^(?=.*[A-Z])(?=.*\d)(?=.*[@$!%*?&])[A-Za-z\d@$!%*?&]{8,}$/
✅ Valid: India@2024!
❌ Invalid: password (no uppercase or digit)

💡Quick Tips
  • Use ^ and $ to match the full string from start to end
  • \d is shorthand for [0-9] — any single digit
  • {n} means exactly n times — use {n,m} for a range
  • [ ] defines a set of allowed characters (e.g., [6-9])
  • ? makes the preceding character or group optional
