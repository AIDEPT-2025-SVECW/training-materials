Spring JPA @GeneratedValue Strategies

Understanding Primary Key Generation in Spring JPA

Explore different strategies for automatically generating primary key values in your JPA entities. Click on each strategy to learn more!

1
GenerationType.IDENTITY
Uses database auto-increment/identity columns for primary key generation.
@Entity public class User { @Id @GeneratedValue(strategy = GenerationType.IDENTITY) private Long id; private String username; // getters and setters... }

Pros

  • Simple to implement
  • Database handles uniqueness
  • Good performance for single inserts
  • No additional tables needed

Cons

  • Poor batch insert performance
  • Database-dependent
  • Cannot get ID before insert
  • Limited to numeric types

Database Support

MySQL PostgreSQL SQL Server SQLite
2
GenerationType.SEQUENCE
Uses database sequences to generate unique primary key values.
@Entity @SequenceGenerator( name = "user_seq_gen", sequenceName = "user_sequence", allocationSize = 50 ) public class User { @Id @GeneratedValue( strategy = GenerationType.SEQUENCE, generator = "user_seq_gen" ) private Long id; // other fields... }

Pros

  • Excellent batch performance
  • Can pre-allocate IDs
  • Database independent (for supported DBs)
  • Configurable increment size

Cons

  • Not all databases support sequences
  • May have gaps in sequence
  • Requires sequence creation
  • Additional database object

Database Support

PostgreSQL Oracle H2 DB2
3
GenerationType.TABLE
Uses a separate database table to store and manage primary key generation.
@Entity @TableGenerator( name = "user_table_gen", table = "id_generator", pkColumnName = "gen_name", valueColumnName = "gen_value", pkColumnValue = "user_id", allocationSize = 50 ) public class User { @Id @GeneratedValue( strategy = GenerationType.TABLE, generator = "user_table_gen" ) private Long id; // other fields... }

Pros

  • Works with any database
  • Highly configurable
  • Can share across entities
  • Portable solution

Cons

  • Poor performance
  • Additional table overhead
  • Potential bottleneck
  • Complex configuration

Database Support

All Databases
4
UUID Generation
Uses Universally Unique Identifiers (UUIDs) for primary keys.
@Entity public class User { @Id @GeneratedValue(strategy = GenerationType.AUTO) @GenericGenerator(name = "uuid2", strategy = "uuid2") private String id; // Alternative using UUID type @Id @GeneratedValue private UUID id; // other fields... }

Pros

  • Globally unique
  • Can generate before insert
  • No database dependency
  • Excellent for distributed systems

Cons

  • Larger storage requirement
  • Not human-readable
  • May impact index performance
  • No natural ordering

Database Support

All Databases
5
GenerationType.AUTO
Lets the JPA provider choose the most appropriate generation strategy.
@Entity public class User { @Id @GeneratedValue(strategy = GenerationType.AUTO) private Long id; // Simplified version (AUTO is default) @Id @GeneratedValue private Long id; // other fields... }

Pros

  • Database-agnostic code
  • Simplest configuration
  • Provider optimizes choice
  • Default behavior

Cons

  • Less control over strategy
  • May not be optimal
  • Unpredictable behavior
  • Hard to optimize

Database Support

Provider Dependent
