Simple SELECT Query
Basic JPQL query to select all users from the database.
@Repository public interface UserRepository extends JpaRepository<User, Long> { @Query("SELECT u FROM User u") List<User> findAllUsers(); }
Returns: List of all User entities
WHERE Clause with Parameters
Query with named parameters for filtering results.
@Repository public interface UserRepository extends JpaRepository<User, Long> { @Query("SELECT u FROM User u WHERE u.age > :minAge AND u.active = :status") List<User> findActiveUsersByAge(@Param("minAge") int minAge, @Param("status") boolean active); }
Returns: List of active users older than specified age
SELECT Specific Fields
Project only specific fields instead of the entire entity.
@Repository public interface UserRepository extends JpaRepository<User, Long> { @Query("SELECT u.username, u.email FROM User u WHERE u.active = true") List<Object[]> findActiveUserInfo(); // Or using constructor expression @Query("SELECT NEW com.example.dto.UserDTO(u.username, u.email) FROM User u") List<UserDTO> findActiveUserDTOs(); }
Returns: Projected fields as Object[] or DTO
ORDER BY and LIMIT
Sorting results and limiting the number of returned records.
@Repository public interface ProductRepository extends JpaRepository<Product, Long> { @Query("SELECT p FROM Product p WHERE p.price > :minPrice ORDER BY p.price DESC") List<Product> findExpensiveProducts(@Param("minPrice") BigDecimal minPrice, Pageable pageable); // Using TOP/FIRST keyword @Query("SELECT p FROM Product p ORDER BY p.price DESC") List<Product> findTop5ByOrderByPriceDesc(Pageable pageable); }
Returns: Sorted products with pagination support
Aggregate Functions
Using COUNT, SUM, AVG, MIN, MAX functions in JPQL.
@Repository public interface OrderRepository extends JpaRepository<Order, Long> { @Query("SELECT COUNT(o) FROM Order o WHERE o.orderDate > :date") long countOrdersAfterDate(@Param("date") LocalDate date); @Query("SELECT AVG(o.total), SUM(o.total), MIN(o.total), MAX(o.total) FROM Order o") Object[] getOrderStatistics(); @Query("SELECT u.username, COUNT(o) FROM User u LEFT JOIN u.orders o GROUP BY u.id, u.username") List<Object[]> getUserOrderCounts(); }
Returns: Aggregated statistics and grouped data
LIKE and Pattern Matching
String pattern matching with wildcards and case sensitivity.
@Repository public interface UserRepository extends JpaRepository<User, Long> { @Query("SELECT u FROM User u WHERE u.username LIKE %:pattern%") List<User> findUsersByUsernamePattern(@Param("pattern") String pattern); @Query("SELECT u FROM User u WHERE LOWER(u.email) LIKE LOWER(CONCAT('%', :domain, '%'))") List<User> findUsersByEmailDomain(@Param("domain") String domain); @Query("SELECT u FROM User u WHERE u.username LIKE :prefix% OR u.username LIKE %:suffix") List<User> findUsersByPrefixOrSuffix(@Param("prefix") String prefix, @Param("suffix") String suffix); }
Returns: Users matching string patterns
IN and BETWEEN Operators
Using IN for multiple values and BETWEEN for ranges.
@Repository public interface ProductRepository extends JpaRepository<Product, Long> { @Query("SELECT p FROM Product p WHERE p.category IN :categories") List<Product> findProductsByCategories(@Param("categories") List<String> categories); @Query("SELECT p FROM Product p WHERE p.price BETWEEN :minPrice AND :maxPrice") List<Product> findProductsByPriceRange(@Param("minPrice") BigDecimal minPrice, @Param("maxPrice") BigDecimal maxPrice); @Query("SELECT p FROM Product p WHERE p.id NOT IN :excludedIds") List<Product> findProductsExcludingIds(@Param("excludedIds") List<Long> excludedIds); }
Returns: Products filtered by lists and ranges
Subqueries
Using subqueries for complex filtering conditions.
@Repository public interface UserRepository extends JpaRepository<User, Long> { @Query("SELECT u FROM User u WHERE u.id IN (SELECT o.user.id FROM Order o WHERE o.total > :amount)") List<User> findUsersWithHighValueOrders(@Param("amount") BigDecimal amount); @Query("SELECT u FROM User u WHERE u.age > (SELECT AVG(u2.age) FROM User u2)") List<User> findUsersOlderThanAverage(); @Query("SELECT u FROM User u WHERE NOT EXISTS (SELECT o FROM Order o WHERE o.user = u)") List<User> findUsersWithoutOrders(); }
Returns: Users based on subquery conditions
JOIN Operations
Different types of JOIN operations in JPQL.
@Repository public interface OrderRepository extends JpaRepository<Order, Long> { // INNER JOIN @Query("SELECT o FROM Order o JOIN o.user u WHERE u.active = true") List<Order> findOrdersForActiveUsers(); // LEFT JOIN @Query("SELECT u, COUNT(o) FROM User u LEFT JOIN u.orders o GROUP BY u") List<Object[]> findUsersWithOrderCount(); // JOIN FETCH (Eager Loading) @Query("SELECT o FROM Order o JOIN FETCH o.user WHERE o.orderDate > :date") List<Order> findOrdersWithUserDetails(@Param("date") LocalDate date); }
Returns: Related entities with various JOIN strategies
Collection Operations
Working with collection properties in entities.
@Repository public interface UserRepository extends JpaRepository<User, Long> { // SIZE function for collection size @Query("SELECT u FROM User u WHERE SIZE(u.orders) > :minOrders") List<User> findUsersWithMinimumOrders(@Param("minOrders") int minOrders); // MEMBER OF for collection membership @Query("SELECT o FROM Order o WHERE :product MEMBER OF o.products") List<Order> findOrdersContainingProduct(@Param("product") Product product); // IS EMPTY for empty collections @Query("SELECT u FROM User u WHERE u.orders IS EMPTY") List<User> findUsersWithNoOrders(); }
Returns: Entities based on collection properties
Complex Relationships
Navigate through multiple entity relationships.
@Repository public interface ProductRepository extends JpaRepository<Product, Long> { // Navigation through multiple relationships @Query("SELECT p FROM Product p JOIN p.orders o JOIN o.user u WHERE u.age BETWEEN :minAge AND :maxAge") List<Product> findProductsOrderedByAgeGroup(@Param("minAge") int minAge, @Param("maxAge") int maxAge); // Complex filtering with relationships @Query("SELECT DISTINCT p FROM Product p JOIN p.orders o WHERE o.orderDate > :date AND o.total > :amount") List<Product> findProductsInRecentHighValueOrders(@Param("date") LocalDate date, @Param("amount") BigDecimal amount); }
Returns: Products based on complex relationship conditions

üí° JPQL Query Tips

  • Always use entity names (not table names) in FROM clauses
  • Use named parameters (:paramName) instead of positional parameters (?1)
  • Remember to use @Param annotation for named parameters
  • Use JOIN FETCH for eager loading to avoid N+1 problems
  • Consider using Pageable for large result sets
  • Use constructor expressions for DTOs: NEW com.example.dto.UserDTO(...)
  • JPQL is case-insensitive for keywords but case-sensitive for entity and property names
  • Use DISTINCT to avoid duplicate results in JOIN queries
