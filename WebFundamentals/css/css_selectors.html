1. Core Selectors: Element, Class, and ID

Quick Concept: These are the three fundamental ways to target HTML elements - by their tag name, their class, or their unique ID.
Think of a college campus. Element selectors are like announcements for all students (broad). Class selectors are like notices for "CS branch students" (specific group). ID selectors are like calling out "Roll Number 2024CS001" (unique individual).

‚öñÔ∏è Which Selector Wins When Multiple Rules Apply?

When the same HTML element is targeted by multiple selectors, the browser uses specificity to decide which style to apply.

üëâ Priority order (from strongest to weakest):

  • ID selector (#id) ‚Äì Highest priority
  • Class selector (.class) ‚Äì Medium priority
  • Element selector (tag) ‚Äì Lowest priority

Simple Example: 

div { background: lightblue; }      /* Element selector */
.topper { background: gold; }       /* Class selector */
#rajesh { background: lightgreen; } /* ID selector */

If all three target the Rajesh element, the background will be lightgreen because ID selector has the highest priority.

üí° Think of it as: ID > Class > Element

üß† More specific selector always wins over a less specific one.

2. Attribute Selectors: The Power of HTML Attributes

Quick Concept: Attribute selectors let you target elements based on their attributes and values, giving you surgical precision in styling.
Like filtering train tickets at a railway station: [type="AC"] finds all AC tickets, [destination*="Mumbai"] finds all tickets going to Mumbai (even if it says "Mumbai Central" or "Mumbai Suburban"), and [class^="First"] finds tickets starting with "First".
üî¨ Interactive Demo: Attribute Selector Types
Attribute Selector Types:
[attr] - Has the attribute
[attr="value"] - Exact match
[attr^="value"] - Starts with
[attr$="value"] - Ends with
[attr*="value"] - Contains
[attr~="value"] - Word match 
/* Has attribute */
[data-validation] { border: 2px solid blue; }

/* Starts with */
[data-validation^="email"] { border: 2px solid red; }

/* Contains */
[data-validation*="required"] { background: #ffe0e0; }

/* Exact match */
[type="text"] { background: #e0f7ff; }

3. Grouping & Universal Selectors (DRY + Global Rules)

Quick Concept: When multiple selectors need the same styles, group them with commas to avoid repetition (Don't Repeat Yourself).
Like a restaurant menu announcement: "Biryani, Pulao, and Fried Rice are 20% off today" instead of saying three separate sentences. One announcement, multiple items!

4. Combining Selectors: Precision Targeting

Quick Concept: Combine selectors to create specific targeting rules using descendant, child, adjacent sibling, and general sibling combinators.
Think of addressing systems: "All houses on MG Road" (descendant), "Direct children of Building A" (child), "The house immediately next to Sharma's house" (adjacent sibling), "All houses after the park" (general sibling).
üìã Combinator Types
Combinator Syntax Meaning Example
Descendant A B B anywhere inside A div p { }
Child A > B B direct child of A div > p { }
Adjacent Sibling A + B B immediately after A h2 + p { }
General Sibling A ~ B All B after A h2 ~ p { }

/* Descendant - All .para inside .container-box (including nested) */
.container-box .para { color: blue; }

/* Child - Only direct .para children of .container-box */
.container-box > .para { color: green; }

/* Adjacent Sibling - First .para immediately after .container-box */
.container-box + .para { color: red; }

/* General Sibling - All .para after .container-box */
.container-box ~ .para { color: purple; }

5. Multiple Selector Combinations

Quick Concept: Chain selectors without spaces for AND logic (element.class), combine with spaces for relationships (parent child).
Like Swiggy filters: "Veg + Fast Delivery + Under ‚Çπ200" (chained - must satisfy ALL). Versus "Restaurants in Koramangala serving Chinese" (combined - relationship between location and cuisine).

.cuisine.veg { background: lightgreen; }
.cuisine.veg.fast { background: gold; }

/* Combining - Relationship between elements (with space) */
.restaurant .cuisine { border: 2px solid blue; }

/* Complex - Chain + Combine */
.restaurant.premium > .cuisine { font-weight: bold; }

6. Pseudo-class & Pseudo-element Selectors

Quick Concept:
  • Pseudo-class selector (:) styles an element based on its state or condition
  • Pseudo-element selector (::) styles a specific part of an element
Pseudo-class is like asking: "Is the switch ON or OFF?"
Pseudo-element is like saying: "Paint only the first letter of this word".
üîπ Pseudo-class Selector
/* Changes style when user interacts */
button:hover {
  background-color: green;
}

input:focus {
  border: 2px solid blue;
}

üí° Styles apply only when the condition is met.

üîπ Pseudo-element Selector
/* Styles part of an element */

  font-size: 30px;
  color: red;
}

p::before {
  content: "üëâ ";
}

üí° Used for styling specific portions or adding decorative content.

Easy Way to Remember:
  • : ‚Üí State (hover, focus, active)
  • :: ‚Üí Part (first letter, before, after)

üìö Quick Reference Summary

Key Takeaways:
  • Element Selector: p { } - Broad targeting
  • Class Selector: .class { } - Group targeting
  • ID Selector: #id { } - Unique targeting
  • Attribute Selectors: [attr], [attr="val"], [attr^="val"], [attr$="val"], [attr*="val"]
  • Grouping: h1, h2, h3 { } - Apply same styles to multiple selectors
  • Combinators: Space (descendant), > (child), + (adjacent), ~ (general sibling)
  • Chaining: element.class1.class2 - Must satisfy ALL (no spaces)
