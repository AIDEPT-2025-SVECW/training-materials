üì¶ Display Property: The Building Blocks

Core Concept: The display property controls how an element behaves in the document flow and how it interacts with other elements.
Think of display types like different types of shops in a Mumbai bazaar:
  • Block: Large shops that occupy an entire row - no other shop can be beside them
  • Inline: Small street vendors who sit side by side in the same row
  • Inline-block: Medium shops that can sit beside each other but have proper structure

üî∑ Block Display

Block elements take full width available and start on a new line. They respect width, height, and all margin/padding.
Block Element 1 (I take full width)
Block Element 2 (I'm on a new line)
Block Element 3 (width: 50%) 
<div style="display: block; width: 100%;">
    I take the full width available
</div>
<div style="display: block;">
    I start on a new line automatically
</div>

/* Common block elements: div, p, h1-h6, section, article */

üîπ Inline Display

Inline elements flow with text, don't break lines, and ignore width/height. Only horizontal padding/margin work.
These are Inline 1 and Inline 2 and Inline 3 elements flowing in text.
Width & Height ignored! 
<span style="display: inline;">I flow with text</span>
<span style="display: inline; width: 200px;">
    Width is ignored!
</span>

/* Common inline elements: span, a, strong, em, img */

Best of both worlds: flows inline but respects width, height, and all padding/margin like block elements.

üéØ Position Property: Controlling Element Placement

Core Concept: Position property determines how an element is positioned in the document and whether it affects the flow of other elements.
Think of CSS Position like students sitting in a classroom üè´
  • Static (Default):
    A student sits on their bench and does not move at all, even if the teacher says ‚Äúmove left or right‚Äù. They follow the normal seating order. (No movement, normal flow)
  • Relative (VERY IMPORTANT):
    A student sits on their bench but shifts a little left or right to be comfortable.
    üëâ The bench is still considered occupied, so no other student can sit there. (Moves from own position, space remains)
  • Absolute:
    A student stands up and sits anywhere inside the classroom wherever the teacher points.
    üëâ Their original bench becomes empty. (Removed from flow, positioned inside parent)
  • Fixed:
    The blackboard or wall clock in the classroom. Even if students move or the class changes, it always stays in the same place. (Fixed to screen)
  • Sticky:
    A student walking normally, but when they reach the poster showing favorite hero, they stop and stay there. (Moves first, then sticks)

üß† Easy Memory Line:
Static sits ‚Ä¢ Relative shifts ‚Ä¢ Absolute escapes ‚Ä¢ Fixed freezes ‚Ä¢ Sticky sticks

üöÇ Static Position (Default)

Default positioning. Element flows naturally in the document. top, right, bottom, left have NO effect.
Static Train (left: 0 has no effect!) 
/* Static is the default - you rarely set it explicitly */

    position: static;
    top: 50px;    /* ‚ùå Ignored */
    left: 100px;  /* ‚ùå Ignored */
}

üîÑ Relative Position

Element moves relative to its ORIGINAL position, but its original space is RESERVED (other elements don't fill it).

üìç Absolute Position

Element is REMOVED from normal flow and positioned relative to its nearest positioned ancestor (parent with position other than static). If none, it's relative to the document body.

üìå Sticky Position

Hybrid of relative and fixed. Element behaves as relative until it reaches a scroll threshold, then becomes fixed WITHIN ITS CONTAINER.

üéØ Key Takeaways

Display Property:

  • Block: Full width, new line, respects all dimensions ‚Üí Use for containers, sections
  • Inline: Flows with text, ignores width/height ‚Üí Use for text styling
  • Inline-Block: Flows inline, respects dimensions ‚Üí Use for buttons, cards, galleries

Position Property:

  • Static: Default flow ‚Üí 95% of your elements
  • Relative: Shifts from original + reserves space ‚Üí Parent for absolute children
  • Absolute: Removed from flow + positioned to ancestor ‚Üí Tooltips, dropdowns
  • Fixed: Stuck to viewport ‚Üí Navigation bars, chat widgets
  • Sticky: Relative then fixed within container ‚Üí Table headers, section titles
üí° Pro Tip: 90% of layout issues come from misunderstanding whether an element is IN or OUT of the document flow. Remember: absolute and fixed are OUT, everything else is IN!
