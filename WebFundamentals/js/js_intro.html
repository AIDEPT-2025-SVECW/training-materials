2Ô∏è‚É£ Brief History of JavaScript

Birth of JS: Created in just 10 days by Brendan Eich in 1995 at Netscape Communications.

Like how India's UPI was created to solve digital payment problems, JavaScript was created to solve the problem of static, boring web pages that couldn't respond to users without refreshing!

1995 - The Birth

Originally named "Mocha", then "LiveScript", finally "JavaScript" (marketing decision to ride Java's popularity)

1997 - Standardization

ECMAScript standard created (ES1) - ensuring all browsers follow same rules

2009 - Node.js Revolution

Ryan Dahl created Node.js, enabling JavaScript to run on servers!

2015 - ES6/ES2015

Major update with modern features: arrow functions, classes, promises, let/const

2025 - Present Day

Most popular language for web development, used by millions of developers worldwide

4Ô∏è‚É£ Linking JavaScript to HTML

Three Ways to Add JS: Just like you can receive money via cash, UPI, or bank transfer - JavaScript can be added to HTML in three ways!

Think of it like ordering food: Inline is like eating at the restaurant, Internal is like home-cooked food, and External is like ordering from Swiggy/Zomato - same food, different delivery methods!

Method 1: Inline JavaScript

Writing JavaScript directly in HTML attributes (not recommended for production)

<!-- Inline JavaScript -->
<button onclick="alert('Namaste! Button clicked!')">
    Click Me
</button>

<!-- Another example -->
<a href="javascript:void(0)" onclick="console.log('Link clicked')">
    Click Link
</a>

Use Case: Quick testing, simple demos

Drawback: Mixes JavaScript with HTML, hard to maintain

Method 2: Internal JavaScript

Writing JavaScript inside <script> tags within the HTML file

<!DOCTYPE html>
<html>
<head>
    <title>Internal JS Example</title>
    <script>
        // JavaScript in the HEAD section
        // Runs before body loads
        console.log("Script in HEAD running");
        
        function greetUser() {
            alert("Hello from internal JavaScript!");
        }
    </script>
</head>
<body>
    <button onclick="greetUser()">Greet</button>
    
    <script>
        // JavaScript in the BODY section
        // Runs after HTML above it loads
        console.log("Script in BODY running");
        
        // Good practice: Put scripts at end of body
        document.addEventListener('DOMContentLoaded', function() {
            console.log("DOM fully loaded!");
        });
    </script>
</body>
</html>

Use Case: Small projects, single-page scripts

Best Practice: Place <script> tags at the end of <body>

Method 3: External JavaScript (RECOMMENDED)

Writing JavaScript in separate .js files and linking them

<!-- HTML file: index.html -->
<!DOCTYPE html>
<html>
<head>
    <title>External JS Example</title>
    <!-- Linking external JS file -->
    <script src="script.js"></script>
    
    <!-- Multiple JS files can be linked -->
    <script src="utils.js"></script>
    <script src="app.js"></script>
</head>
<body>
    <h1>My Website</h1>
    
    <!-- Modern approach: defer attribute -->
    <script src="main.js" defer></script>
    
    <!-- For async loading -->
    <script src="analytics.js" async></script>
</body>
</html>
/* JavaScript file: script.js */
// All your JavaScript code goes here
console.log("External JS file loaded!");

function calculateGST(price) {
    const gst = price * 0.18;
    return price + gst;
}

// Event listeners
document.addEventListener('DOMContentLoaded', function() {
    console.log("Ready to rock!");
});

Use Case: Production applications, large projects

Benefits: Reusable, cacheable, organized, maintainable

Understanding defer and async Attributes:

<!-- Normal script: Blocks HTML parsing -->
<script src="normal.js"></script>

<!-- Defer: Downloads in parallel, executes after HTML parsing -->
<script src="defer.js" defer></script>

<!-- Async: Downloads in parallel, executes immediately when ready -->
<script src="async.js" async></script>

üö´ Normal Script

Blocks HTML parsing until script downloads and executes

‚è∏Ô∏è defer

Downloads while HTML parses, executes after parsing completes. Maintains order.

‚ö° async

Downloads while HTML parses, executes immediately when ready. Order not guaranteed.
