1. Variables: var, let, const

Quick Summary: Variables are containers for storing data. JavaScript has three ways to declare them, each with different scoping and mutability rules.
Analogy: Think of variables like parking spots in India:
  • var = Public parking (anyone can access from anywhere in the function)
  • let = Reserved parking with specific zone (block-scoped, can be reassigned)
  • const = Permanently reserved spot (block-scoped, cannot be reassigned)

Comparison Table

Feature var let const
Scope Function-scoped Block-scoped Block-scoped
Reassignment ‚úÖ Yes ‚úÖ Yes ‚ùå No
Hoisting ‚úÖ Yes (initialized as undefined) ‚úÖ Yes (temporal dead zone) ‚úÖ Yes (temporal dead zone)
Redeclaration ‚úÖ Allowed ‚ùå Not allowed ‚ùå Not allowed

Code Examples

// var - Function scoped
function varExample() {
    if (true) {
        var city = "Mumbai";
    }
    console.log(city); // "Mumbai" - accessible outside block!
}

// let - Block scoped
function letExample() {
    if (true) {
        let city = "Delhi";
    }
    console.log(city); // Error: city is not defined
}

// const - Block scoped, cannot reassign
const PI = 3.14159;
PI = 3.14; // Error: Assignment to constant variable

// But object properties can be modified
const student = { name: "Raj" };
student.name = "Priya"; // This works!
student = {}; // Error: Assignment to constant variable

2. Strings

Quick Summary: Strings are sequences of characters used to represent text. JavaScript offers multiple ways to create and manipulate strings.
Analogy: Strings are like WhatsApp messages - you can send them in different formats:
  • Single quotes: 'Regular text message'
  • Double quotes: "Same as single quotes"
  • Template literals: `Message with ${variables} - like forwarding with personalization!`

Code Examples

// Three ways to create strings
let name1 = 'Amit';  // Single quotes
let name2 = "Priya"; // Double quotes
let name3 = `Rahul`; // Template literals (backticks)

// Template literals - The powerful one!
let firstName = "Arjun";
let city = "Bangalore";
let greeting = `Hello ${firstName} from ${city}!`; // Hello Arjun from Bangalore!

// Multi-line strings with template literals
let address = `123, MG Road
Bangalore, Karnataka
560001`;

// String methods
let college = "IIT Bombay";
college.length;           // 10
college.toLowerCase();    // "iit bombay"
college.toUpperCase();    // "IIT BOMBAY"
college.includes("IIT");  // true
college.slice(0, 3);      // "IIT"
college.split(" ");       // ["IIT", "Bombay"]

// String concatenation
let fullName = "Virat" + " " + "Kohli"; // Old way
let fullName2 = `Virat Kohli`;          // Modern way

3. Numbers

Quick Summary: JavaScript has only ONE number type. All numbers are 64-bit floating point (unlike Java/C++ with int, float, double).
Analogy: JavaScript numbers are like UPI payments in India - whether you're paying ‚Çπ10 or ‚Çπ10.50, it's all handled the same way. No need to worry about "integer rupees" vs "decimal paise" separately!

Code Examples

// All these are the same type: number
let price = 100;              // Integer-like
let discount = 12.5;          // Decimal
let population = 1.4e9;       // Scientific notation (1.4 billion)
let negative = -50;           // Negative

// Special numeric values
let infinity = Infinity;      // Division by zero
let notANumber = NaN;         // Invalid math operation
let result = 0 / 0;           // NaN

// Number methods and operations
let rupees = 1234.56789;
rupees.toFixed(2);            // "1234.57" (rounds to 2 decimals)
rupees.toPrecision(5);        // "1234.6" (5 significant digits)

// Parsing strings to numbers
parseInt("100");              // 100
parseInt("100.99");           // 100 (ignores decimal)
parseFloat("100.99");         // 100.99
Number("100");                // 100

// Math operations
Math.round(4.7);              // 5
Math.ceil(4.1);               // 5 (round up)
Math.floor(4.9);              // 4 (round down)
Math.random();                // Random between 0 and 1
Math.max(10, 20, 30);         // 30
Math.min(10, 20, 30);         // 10
Math.pow(2, 3);               // 8 (2 to the power 3)
Math.sqrt(16);                // 4

// Checking for NaN and Infinity
isNaN(NaN);                   // true
isFinite(100);                // true
isFinite(Infinity);           // false

4. Arrays

Quick Summary: Arrays are ordered collections that can hold multiple values of any type. They're zero-indexed and have powerful built-in methods.
Analogy: Arrays are like train compartments in Indian Railways:
  • Each compartment (index) has a number: [0], [1], [2]...
  • You can put different passengers (values) in each compartment
  • You can add more compartments (push), remove them (pop), or even insert in between!

Code Examples

// Creating arrays
let cities = ["Mumbai", "Delhi", "Bangalore"];
let numbers = [1, 2, 3, 4, 5];
let mixed = [1, "text", true, null, {name: "Raj"}]; // Can mix types!

// Accessing elements (zero-indexed)
cities[0];           // "Mumbai"
cities[1];           // "Delhi"
cities[2];           // "Bangalore"
cities[3];           // undefined (doesn't exist)

// Array properties
cities.length;       // 3

// Adding elements
cities.push("Chennai");        // Add to end: ["Mumbai", "Delhi", "Bangalore", "Chennai"]
cities.unshift("Kolkata");     // Add to start: ["Kolkata", "Mumbai", ...]

// Removing elements
cities.pop();                  // Remove from end, returns "Chennai"
cities.shift();                // Remove from start, returns "Kolkata"

// Finding elements
cities.indexOf("Delhi");       // 1 (index position)
cities.includes("Mumbai");     // true

// Slicing and splicing
let iits = ["Bombay", "Delhi", "Madras", "Kanpur", "Kharagpur"];
iits.slice(1, 3);             // ["Delhi", "Madras"] (doesn't modify original)
iits.splice(2, 1, "Chennai"); // Replace "Madras" with "Chennai"

// Iterating
let fruits = ["Mango", "Banana", "Apple"];

// Traditional for loop
for (let i = 0; i < fruits.length; i++) {
    console.log(fruits[i]);
}

// Modern forEach
fruits.forEach(function(fruit) {
    console.log(fruit);
});

// Array methods for transformation
let prices = [100, 200, 300];
prices.map(p => p * 0.9);     // [90, 180, 270] - 10% discount
prices.filter(p => p > 150);  // [200, 300] - prices above 150
prices.reduce((sum, p) => sum + p, 0); // 600 - total sum

5. Objects

Quick Summary: Objects are collections of key-value pairs. They're fundamental to JavaScript and represent real-world entities with properties and behaviors.
Analogy: Objects are like Aadhaar cards - each person (object) has properties like name, age, address (key-value pairs). You can access any property directly using its key, just like looking up specific information on your Aadhaar card!

Code Examples

// Creating objects - Multiple ways

// 1. Object literal (most common)
let student = {
    name: "Raj Kumar",
    age: 21,
    college: "IIT Delhi",
    branch: "Computer Science",
    cgpa: 8.5
};

// 2. Using new Object()
let professor = new Object();
professor.name = "Dr. Sharma";
professor.department = "CSE";

// Accessing properties - Two ways
student.name;              // "Raj Kumar" (dot notation)
student["age"];            // 21 (bracket notation)

// Bracket notation is useful for dynamic keys
let property = "college";
student[property];         // "IIT Delhi"

// Adding new properties
student.email = "raj@iit.ac.in";
student.hobbies = ["cricket", "coding"];

// Modifying properties
student.cgpa = 9.0;

// Deleting properties
delete student.branch;

// Nested objects
let person = {
    name: "Priya",
    address: {
        street: "MG Road",
        city: "Bangalore",
        pincode: 560001
    },
    contact: {
        mobile: "9876543210",
        email: "priya@email.com"
    }
};

// Accessing nested properties
person.address.city;           // "Bangalore"
person["contact"]["mobile"];   // "9876543210"

// Object methods
let car = {
    brand: "Tata",
    model: "Nexon",
    year: 2024,
    // Method inside object
    getInfo: function() {
        return this.brand + " " + this.model;
    }
};

car.getInfo();  // "Tata Nexon"

// Useful Object methods
Object.keys(student);      // ["name", "age", "college", "cgpa", "email", "hobbies"]
Object.values(student);    // ["Raj Kumar", 21, "IIT Delhi", 9.0, "raj@iit.ac.in", Array(2)]
Object.entries(student);   // [["name", "Raj Kumar"], ["age", 21], ...]

// Checking if property exists
"name" in student;         // true
student.hasOwnProperty("age"); // true

‚ú® Object Destructuring (Modern Way to Extract Properties)

Quick Summary: Destructuring allows us to extract specific properties from an object into variables directly. It makes code shorter, cleaner, and more readable.
Analogy: Destructuring is like taking only what you need from your Aadhaar card. Instead of carrying the whole card everywhere, you extract only:
  • Name
  • City
  • Mobile Number
Similarly, destructuring picks only required properties from an object. 

// Example Object
let student = {
    name: "Amit Sharma",
    age: 22,
    college: "NIT Trichy",
    branch: "Mechanical"
};

// Traditional Way
let studentName = student.name;
let studentCollege = student.college;

// Destructuring Way (Modern & Cleaner)
let { name, college } = student;

console.log(name);     // "Amit Sharma"
console.log(college);  // "NIT Trichy"

// Renaming while destructuring
let { name: fullName } = student;
console.log(fullName); // "Amit Sharma"

// Default value
let { semester = 6 } = student;
console.log(semester); // 6 (used because not present in object)

// Nested destructuring
let person = {
    name: "Priya",
    address: {
        city: "Bangalore",
        pincode: 560001
    }
};

let { address: { city } } = person;
console.log(city); // "Bangalore"

// Destructuring in Function Parameters
function printStudent({ name, branch }) {
    console.log(name + " studies " + branch);
}

printStudent(student);

6. Operators & Expressions

Quick Summary: Operators perform operations on values. JavaScript has arithmetic, comparison, logical, assignment, and special operators.
Analogy: Operators are like traffic signals and road signs in India:
  • Arithmetic (+, -, *, /): Like meter readings in auto-rickshaws
  • Comparison (===, !==): Like checking if two Aadhaar numbers match
  • Logical (&&, ||): Like "AND" gate at railway crossings (both conditions must be true)

Operator Categories

A. Arithmetic Operators

let a = 10, b = 3;

a + b;    // 13 (Addition)
a - b;    // 7  (Subtraction)
a * b;    // 30 (Multiplication)
a / b;    // 3.333... (Division)
a % b;    // 1  (Modulo - remainder)
a ** b;   // 1000 (Exponentiation - 10^3)

// Increment and Decrement
let count = 5;
count++;  // 6 (post-increment)
++count;  // 7 (pre-increment)
count--;  // 6 (post-decrement)
--count;  // 5 (pre-decrement)

B. Comparison Operators

// Equality operators (IMPORTANT!)
5 == "5";     // true  (loose equality - converts types)
5 === "5";    // false (strict equality - checks type too)
5 != "5";     // false (loose inequality)
5 !== "5";    // true  (strict inequality)

// ‚ö†Ô∏è Always use === and !== to avoid bugs!

// Relational operators
10 > 5;       // true
10 < 5;       // false
10 >= 10;     // true
10 <= 5;      // false

// Examples with prices
let price = 599;
let budget = 500;

price > budget;      // true
price <= budget;     // false

C. Logical Operators

// AND (&&) - Both must be true
let hasLicense = true;
let hasVehicle = true;
hasLicense && hasVehicle;  // true (can drive!)

let age = 17;
let hasID = true;
(age >= 18) && hasID;      // false (age fails)

// OR (||) - At least one must be true
let hasCard = false;
let hasCash = true;
hasCard || hasCash;        // true (can pay!)

// NOT (!) - Inverts boolean
let isRaining = false;
!isRaining;                // true (not raining)

// Combining logical operators
let temperature = 35;
let isSummer = true;
let isHot = (temperature > 30) && isSummer;  // true

// Short-circuit evaluation
let userName = "";
let displayName = userName || "Guest";  // "Guest" (because userName is falsy)

D. Assignment Operators

let score = 100;

score = 200;      // Basic assignment
score += 50;      // score = score + 50  ‚Üí 250
score -= 30;      // score = score - 30  ‚Üí 220
score *= 2;       // score = score * 2   ‚Üí 440
score /= 4;       // score = score / 4   ‚Üí 110
score %= 30;      // score = score % 30  ‚Üí 20

// Practical example: Shopping cart
let cartTotal = 0;
cartTotal += 499;  // Added item ‚Çπ499
cartTotal += 299;  // Added item ‚Çπ299
cartTotal -= 50;   // Applied discount ‚Çπ50
// cartTotal is now ‚Çπ748

E. Special Operators

// typeof - Check data type
typeof 42;              // "number"
typeof "hello";         // "string"
typeof true;            // "boolean"
typeof [1, 2, 3];       // "object" (arrays are objects!)
typeof {name: "Raj"};   // "object"
typeof undefined;       // "undefined"

// Ternary operator (? :) - Short if-else
let age = 20;
let status = (age >= 18) ? "Adult" : "Minor";  // "Adult"

// Same as:
// if (age >= 18) {
//     status = "Adult";
// } else {
//     status = "Minor";
// }

// Nullish coalescing (??) - Default values
let userInput = null;
let value = userInput ?? "Default";  // "Default"

// String concatenation with +
let greeting = "Hello" + " " + "World";  // "Hello World"
let total = "Price: ‚Çπ" + 500;            // "Price: ‚Çπ500"

üíé Bonus: Type Conversion & Coercion

Quick Summary: JavaScript can automatically convert types (coercion) or you can manually convert them. Understanding this prevents common bugs!
Analogy: Type conversion is like currency exchange at airports - you can explicitly exchange USD to INR (explicit conversion) or sometimes the shopkeeper automatically converts it for you (implicit coercion)!

Code Examples

// Implicit Coercion (JavaScript does it automatically)
"5" + 2;          // "52" (number converted to string)
"5" - 2;          // 3 (string converted to number)
"5" * "2";        // 10 (both strings converted to numbers)
true + 1;         // 2 (true becomes 1)
false + 1;        // 1 (false becomes 0)

// ‚ö†Ô∏è This can cause bugs!
let price = "100";
let tax = 18;
price + tax;      // "10018" - WRONG! String concatenation

// Explicit Conversion (You control it)
Number("100");    // 100
String(100);      // "100"
Boolean(1);       // true
Boolean(0);       // false

// Fix the bug above
Number(price) + tax;  // 118 - CORRECT!

// Falsy values in JavaScript (convert to false)
Boolean(0);           // false
Boolean("");          // false
Boolean(null);        // false
Boolean(undefined);   // false
Boolean(NaN);         // false

// Everything else is truthy!
Boolean(1);           // true
Boolean("hello");     // true
Boolean([]);          // true
Boolean({});          // true
