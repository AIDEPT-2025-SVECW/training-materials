üñºÔ∏è The IMG Tag - Beyond Basics

The <img> tag embeds images into web pages. It's a self-closing tag that requires proper attributes for functionality and accessibility.
Think of the <img> tag like a photo frame in your home. The frame (img tag) is empty until you specify which photo (src) to display, and you might add a label (alt) for when guests can't see the picture clearly.

Core IMG Attributes

<!-- Basic Image -->
<img src="path/to/image.jpg" alt="Description of image">

<!-- Image with Dimensions -->
<img src="taj-mahal.jpg" 
     alt="Taj Mahal at sunrise" 
     width="600" 
     height="400">

<!-- Image with Loading Optimization -->
<img src="large-image.jpg" 
     alt="High resolution photo" 
     loading="lazy"
     decoding="async">

‚ö° Image Performance: loading & decoding

Modern browsers provide loading and decoding attributes to control when an image is loaded and how it is processed, improving page speed and user experience.
Imagine images like students entering a classroom. loading decides when students enter the room, while decoding decides how calmly or urgently they settle down without disturbing the class.

1Ô∏è‚É£ loading Attribute ‚Äî When should the image load?

The loading attribute tells the browser when to download an image.

Value Meaning Best Use Case
lazy Loads image only when it comes near the viewport Images below the fold, galleries, long pages
eager Loads image immediately Hero images, banners
auto Browser decides automatically Default behavior 
<img src="gallery-photo.jpg"
     alt="Gallery image"
     loading="lazy">
Performance Tip Lazy loading reduces initial page load time and saves bandwidth.

2Ô∏è‚É£ decoding Attribute ‚Äî How should the image be processed?

The decoding attribute tells the browser how to decode the image after downloading. Decoding means converting image data into pixels on the screen.

Value Behavior Impact
async Decodes image in the background Does not block page rendering
sync Decodes immediately May delay text/layout rendering
auto Browser decides Default behavior 
<img src="large-image.jpg"
     alt="High resolution image"
     decoding="async">
UX Tip Async decoding keeps the page responsive while images load.

3Ô∏è‚É£ Using loading and decoding Together

Combining both attributes gives the best performance for large images that are not immediately visible. 

<img src="large-image.jpg"
     alt="High resolution photo"
     loading="lazy"
     decoding="async">

What happens here?

  • ‚úî Image loads only when needed
  • ‚úî Image decoding does not block page rendering
  • ‚úî Faster initial page load

‚úî Best Practices

  • Use loading="lazy" for non-critical images
  • Avoid lazy loading for above-the-fold images
  • Use decoding="async" for large images
  • Always specify alt text for accessibility

üé¨ Video Element - Rich Media Control

The <video> tag provides native browser support for video playback without plugins, with extensive JavaScript API control.
The video element is like a TV with a remote control. The <video> tag is the TV itself, attributes are preset buttons (volume, channel), and JavaScript is the full remote that lets you control everything programmatically.

Video Tag Structure

<!-- Basic Video with Controls -->
<video width="640" height="360" controls>
    <source src="movie.mp4" type="video/mp4">
    <source src="movie.webm" type="video/webm">
    Your browser doesn't support video playback.
</video>

<!-- Advanced Video with Attributes -->
<video width="800" height="450" 
       controls 
       autoplay 
       muted 
       loop 
       poster="thumbnail.jpg"
       preload="metadata">
    <source src="video.mp4" type="video/mp4">
    <source src="video.webm" type="video/webm">
    <track src="subtitles-en.vtt" 
           kind="subtitles" 
           srclang="en" 
           label="English">
</video>

üéÆ Interactive: Video Player Builder

Video Attributes Explained:

controls Shows play/pause/volume controls
autoplay Starts playing automatically
muted Starts with no sound
loop Repeats continuously
poster Thumbnail before playback
preload none | metadata | auto

Video Format Support

Why multiple <source> tags? Different browsers support different formats. The browser picks the first format it can play. 

<video controls>
    <source src="video.mp4" type="video/mp4">      <!-- Chrome, Safari, Edge -->
    <source src="video.webm" type="video/webm">    <!-- Firefox, Chrome -->
    <source src="video.ogv" type="video/ogg">      <!-- Older Firefox -->
    Fallback text for ancient browsers
</video>

üéµ Audio Element - Sound Integration

The <audio> tag works similarly to video but for sound files. It's perfect for background music, podcasts, or sound effects.
If <video> is a TV, then <audio> is a radio. Same control mechanisms, but focused only on sound without the visual component.

Audio Tag Implementation

<!-- Basic Audio Player -->
<audio controls>
    <source src="song.mp3" type="audio/mpeg">
    <source src="song.ogg" type="audio/ogg">
    Your browser doesn't support audio.
</audio>

<!-- Background Music (autoplay must be muted) -->
<audio autoplay loop muted>
    <source src="background.mp3" type="audio/mpeg">
</audio>

<!-- Preloaded Audio for Quick Play -->
<audio id="notification" preload="auto">
    <source src="beep.mp3" type="audio/mpeg">
</audio>

<script>
    // Play audio on button click
    document.getElementById('playBtn').onclick = function() {
        document.getElementById('notification').play();
    };
</script>

ü™ü IFrame - Embedding External Content

<iframe> creates a nested browsing context, essentially embedding another webpage within your page. Think embedded YouTube videos, Google Maps, etc.
An iframe is like a window in your house that shows your neighbor's house. You control the window's size and position, but the content inside belongs to someone else (another website).

IFrame Syntax & Security

<!-- Basic IFrame -->
<iframe src="https://www.example.com" 
        width="800" 
        height="600">
</iframe>

<!-- YouTube Embed -->
<iframe width="560" height="315" 
        src="https://www.youtube.com/embed/VIDEO_ID" 
        title="YouTube video player" 
        frameborder="0" 
        allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture" 
        allowfullscreen>
</iframe>

<!-- Google Maps Embed -->
<iframe src="https://www.google.com/maps/embed?pb=..." 
        width="600" 
        height="450" 
        style="border:0;" 
        allowfullscreen="" 
        loading="lazy" 
        referrerpolicy="no-referrer-when-downgrade">
</iframe>

<!-- Secure IFrame with Sandbox -->
<iframe src="untrusted.html" 
        sandbox="allow-scripts allow-same-origin">
</iframe>

üéÆ Interactive: IFrame Generator

IFrame Security Attributes

sandbox attribute restricts iframe capabilities:

  • allow-scripts - Permits JavaScript execution
  • allow-same-origin - Treats content as same origin
  • allow-forms - Allows form submission
  • allow-popups - Permits popup windows
  • allow-top-navigation - Can navigate top-level window 
<!-- Restricted IFrame (no scripts, isolated origin) -->
<iframe src="untrusted.html" sandbox></iframe>

<!-- IFrame with specific permissions -->
<iframe src="form.html" sandbox="allow-forms allow-scripts"></iframe>

üìÅ File Paths - Absolute vs Relative

File paths tell the browser where to find resources. Understanding path types is crucial for working with images, videos, and embedded content.
File paths are like directions to your friend's house. Absolute path: "123 MG Road, Bangalore, Karnataka 560001" (complete address). Relative path: "two houses down from mine" (relative to your current location).

<!-- Root-relative (starts from domain root) -->

‚ôø Accessibility - Alt Text & Beyond

Accessibility ensures all users, including those with disabilities, can access your content. Alt text is the foundation of image accessibility.
Alt text is like describing a painting to someone over the phone. You need to convey the essential information and context without them seeing it. For a "Submit" button icon, you'd say "Submit" not "green checkmark image."

Writing Good Alt Text

<!-- POOR Alt Text ‚ùå -->
<img src="photo.jpg" alt="image">
<img src="chart.png" alt="chart">
<img src="logo.png" alt="company logo picture">

<!-- GOOD Alt Text ‚úÖ -->
<img src="taj-mahal.jpg" 
     alt="Taj Mahal white marble mausoleum at sunrise">

<img src="sales-chart.png" 
     alt="Bar chart showing 40% increase in sales from Q1 to Q2 2024">

<img src="acme-logo.png" 
     alt="Acme Corporation">  <!-- Skip "logo" -->

<!-- Decorative Image (empty alt) -->
<img src="decorative-divider.png" alt="">

<!-- Complex Image with Long Description -->
<img src="infographic.png" 
     alt="Employee satisfaction survey results" 
     longdesc="detailed-analysis.html">

üéÆ Interactive: Alt Text Evaluator

Alt Text Decision Tree

Is the image informative or decorative?
  • Informative ‚Üí Provide descriptive alt text
  • Decorative ‚Üí Use empty alt="" (not omit attribute)
  • Functional (button/link) ‚Üí Describe the action (e.g., "Search", "Close menu")
  • Complex (chart/graph) ‚Üí Brief alt + link to full description

‚ö° Performance Optimization

Image Optimization Techniques

<!-- Lazy Loading (load when near viewport) -->
<img src="image.jpg" 
     alt="Description" 
     loading="lazy">

<!-- Async Decoding (don't block rendering) -->
<img src="large-image.jpg" 
     alt="Description" 
     decoding="async">

<!-- Preload Critical Images -->
<link rel="preload" as="image" href="hero-image.jpg">

<!-- Modern Format with Fallback -->
<picture>
    <source srcset="image.webp" type="image/webp">
    <source srcset="image.jpg" type="image/jpeg">
    <img src="image.jpg" alt="Description">
</picture>

<!-- Inline Small Images (Base64) -->
<img src="data:image/svg+xml;base64,PHN2ZyB3aWR0aD0i..." 
     alt="Small icon">

Video Optimization

<!-- Lazy Load Videos -->
<video controls preload="none" poster="thumbnail.jpg">
    <source src="video.mp4" type="video/mp4">
</video>

<!-- Preload Metadata Only -->
<video controls preload="metadata">
    <source src="video.mp4" type="video/mp4">
</video>

<!-- Preload Full Video (use sparingly) -->
<video controls preload="auto">
    <source src="video.mp4" type="video/mp4">
</video>
preload Value Behavior Best For
none Nothing loads until user clicks play Pages with many videos
metadata Loads duration, dimensions, first frame Most common use case
auto Loads entire video Critical videos user will likely play

üìö Key Takeaways

Images

  • Always include descriptive alt text
  • Use appropriate format: JPEG (photos), PNG (graphics), WebP (modern)
  • Implement lazy loading for below-fold images
  • Use srcset for responsive images

Video & Audio

  • Provide multiple source formats for browser compatibility
  • Include captions/subtitles for accessibility
  • Use poster for video thumbnails
  • Set appropriate preload values

IFrames

  • Use for embedding third-party content
  • Implement sandbox for security
  • Specify dimensions to prevent layout shifts
  • Use loading="lazy" for off-screen iframes

File Paths

  • Use relative paths for portability
  • Root-relative (/) for consistency across pages
  • Absolute URLs for external resources
  • Understand ../ for navigating up directories
