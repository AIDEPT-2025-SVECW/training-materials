Spring Boot @Controller vs @RestController

@Controller

@Controller public class UserController { @GetMapping("/user") public String getUser(Model model) { model.addAttribute("user", new User()); return "user-view"; // Returns view name } @GetMapping("/api/user") @ResponseBody // Need this for JSON public User getUserJson() { return new User("John", "Doe"); } }
  • Purpose: Handles web requests and returns views
  • Response: Returns view names (HTML pages)
  • JSON Response: Requires @ResponseBody annotation
  • Use Case: Traditional MVC web applications
  • Content Type: text/html by default

@RestController

@RestController public class UserRestController { @GetMapping("/api/users") public List<User> getUsers() { return Arrays.asList( new User("John", "Doe"), new User("Jane", "Smith") ); // Automatic JSON conversion } @PostMapping("/api/users") public ResponseEntity<User> createUser(@RequestBody User user) { return ResponseEntity.ok(user); } }
  • Purpose: Handles REST API requests
  • Response: Returns data directly (JSON/XML)
  • JSON Response: Automatic without @ResponseBody
  • Use Case: REST APIs and web services
  • Content Type: application/json by default
Key Insight: @RestController = @Controller + @ResponseBody on every method

Key Differences Summary

Return Type

@Controller: View name (String) or ModelAndView

@RestController: Data objects (automatically serialized)

Response Format

@Controller: HTML pages

@RestController: JSON/XML data

@ResponseBody

@Controller: Required for JSON responses

@RestController: Automatic on all methods

Use Case

@Controller: Traditional web apps with views

@RestController: REST APIs and microservices
