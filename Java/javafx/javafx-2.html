Core Idea: UI Controls are pre-built JavaFX components that handle user input and display information. Each control extends the Control class and comes with built-in behavior.
Think of JavaFX Nodes as the actors and props in a stage performance. The Stage is the theater itself, the Scene is the current act being performed, and the Layout is the stage director who decides where each actor or prop should stand. Each Node ‚Äî like a Button, Label, or Image ‚Äî plays a role in the act: Buttons perform actions, Labels deliver lines (text), and Images set the mood or background. Change the Layout, and the same actors (Nodes) rearrange their positions to suit the new choreography!

// JavaFX TextField with validation
TextField mobileField = new TextField();
mobileField.setPromptText("Enter 10-digit mobile number");

// Add change listener for real-time validation
mobileField.textProperty().addListener((observable, oldValue, newValue) -> {
    if (!newValue.matches("\\d*")) {
        mobileField.setText(oldValue); // Only allow digits
    }
    if (newValue.length() > 10) {
        mobileField.setText(oldValue); // Limit to 10 digits
    }
});
TextField is like a form at a government office. The PromptText is like the helper text ("Enter Aadhar Number"), the text validation is like the clerk checking if you filled it correctly, and the maxLength is like the fixed number of boxes on the form.

‚ö° Event Handling - Making Controls Interactive

Core Idea: Events are user actions (clicks, typing, mouse movements) that trigger code execution. JavaFX uses an event-driven architecture where you register handlers to respond to specific events.
Event handling is like a doorbell system. The doorbell (UI Control) is pressed (Event occurs), which sends a signal (Event object) to the buzzer (Event Handler), which then rings (executes your code). You can have multiple doorbells (controls) connected to the same buzzer (handler) or different buzzers.

2. Lambda Expressions vs Anonymous Inner Classes

Interactive Demo: Two Ways to Handle Events

Click a button to see code style... 
// Method 1: Lambda Expression (Concise & Modern)
submitButton.setOnAction(e -> {
    String text = textField.getText();
    label.setText("Submitted: " + text);
});

// Method 2: Anonymous Inner Class (Verbose but explicit)
submitButton.setOnAction(new EventHandler<ActionEvent>() {
    @Override
    public void handle(ActionEvent e) {
        String text = textField.getText();
        label.setText("Submitted: " + text);
    }
});

3. Event Propagation - Bubbling & Capturing

Interactive Demo: Event Flow

Click the button to see event flow...

Notice how events bubble up from child to parent! 

// JavaFX Event Propagation
VBox parent = new VBox();
HBox child = new HBox();
Button button = new Button("Click");

// Parent handler (fires second in bubbling phase)
parent.setOnMouseClicked(e -> {
    System.out.println("Parent clicked");
});

// Child handler (fires first)
button.setOnAction(e -> {
    System.out.println("Button clicked");
    // e.consume(); // Stop propagation to parent
});
Event propagation is like a student raising hand in class (Button), then the class monitor notices (Child container), then the teacher sees (Parent container). If the class monitor handles it immediately (consume event), the teacher doesn't need to respond.

4. Practical Example: Login Form with Validation

// Complete JavaFX Login Form
TextField usernameField = new TextField();
PasswordField passwordField = new PasswordField();
CheckBox rememberCheckBox = new CheckBox("Remember me");
Button loginButton = new Button("Login");
Label statusLabel = new Label();

// Disable button initially
loginButton.setDisable(true);

// Real-time validation
ChangeListener<String> validator = (obs, old, newVal) -> {
    boolean valid = usernameField.getText().length() >= 4 
                 && passwordField.getText().length() >= 6;
    loginButton.setDisable(!valid);
};

usernameField.textProperty().addListener(validator);
passwordField.textProperty().addListener(validator);

// Login action
loginButton.setOnAction(e -> {
    String user = usernameField.getText();
    String pass = passwordField.getText();
    boolean remember = rememberCheckBox.isSelected();
    
    if (authenticateUser(user, pass)) {
        statusLabel.setText("Welcome, " + user + "!");
        if (remember) {
            saveCredentials(user);
        }
    } else {
        statusLabel.setText("Invalid credentials!");
    }
});

5. Event Handling Best Practices

  • Use Lambda Expressions: Cleaner and more readable for simple handlers
  • Separate Business Logic: Don't put heavy processing inside event handlers
  • Validate Early: Provide real-time feedback, don't wait for submit
  • Consume Events Appropriately: Stop propagation when needed with consume()
  • Use Property Listeners: For dynamic updates based on control state changes

Teaching Tips:

  1. Live Demo First: Show each control in Scene Builder before coding. Let students see the visual before the syntax.
  2. Common Mistake Alert: Students often forget to add controls to layout panes or forget to call show() on Stage. Have a checklist ready.
