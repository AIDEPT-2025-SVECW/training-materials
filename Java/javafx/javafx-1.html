🎨 Introduction to JavaFX

Building Rich Client Applications with Modern UI Framework

1. What is JavaFX?

Concept: JavaFX is a modern Java framework for building rich, desktop applications with sophisticated UIs, replacing the older Swing framework with better graphics, CSS styling, and FXML support.
Think of JavaFX as the "React.js for desktop applications" - just like React gives you components and a virtual DOM for web apps, JavaFX gives you nodes and a scene graph for desktop apps. Your UI is a tree of components!
Feature Swing (Old) JavaFX (Modern)
UI Design Code-based only Code + FXML (like HTML)
Styling Java code styling CSS styling support
Graphics Basic 2D Advanced 2D + 3D + Effects
Multimedia Limited Built-in audio/video support
Animation Manual timer-based Built-in animation API 
// Basic JavaFX Application Structure
import javafx.application.Application;
import javafx.scene.Scene;
import javafx.scene.control.Button;
import javafx.scene.layout.StackPane;
import javafx.stage.Stage;

public class HelloJavaFX extends Application {
    
    @Override
    public void start(Stage primaryStage) {
        // Create a button - this is a Node
        Button btn = new Button("नमस्ते JavaFX!");
        btn.setOnAction(e -> System.out.println("Button clicked!"));
        
        // Create a layout - also a Node
        StackPane root = new StackPane();
        root.getChildren().add(btn);
        
        // Create a Scene - contains the node tree
        Scene scene = new Scene(root, 300, 250);
        
        // Set up the Stage - your application window
        primaryStage.setTitle("My First JavaFX App");
        primaryStage.setScene(scene);
        primaryStage.show();
    }
    
    public static void main(String[] args) {
        launch(args);
    }
}

2. Core Architecture: Stage → Scene → LayOut → Nodes

Concept: JavaFX follows a theatrical hierarchy: an Application opens a Stage (the window). The Stage shows a Scene (the content area). Inside a Scene you place a Layout (VBox, HBox, GridPane, BorderPane, etc.) which manages positioning. The Layout then contains the actual Nodes (buttons, labels, text fields, images) — every visible thing in JavaFX is a Node arranged by a Layout in the Scene graph.
Imagine a film production: the Stage is the studio building, the Scene is the set built for a particular shot, the Layout is the floor plan or stage manager who positions furniture and actors, and the Nodes are the actors, props and lights. Change the Layout (floor plan) and the same actors/props (Nodes) will be arranged differently within the Scene — just like switching from a sitting arrangement to a standing formation on set.

// Understanding the Hierarchy
public void start(Stage stage) {
    // STAGE - The top-level window
    stage.setTitle("Mumbai Local Train Schedule");
    
    // SCENE - Container for your UI tree
    VBox root = new VBox(10); // Root Node (a layout)
    Scene scene = new Scene(root, 400, 300);
    
    // NODES - Your UI elements (children of root)
    Label header = new Label("Western Line Trains");
    Button refreshBtn = new Button("Refresh Schedule");
    ListView<String> trainList = new ListView<>();
    
    // Build the Scene Graph (tree structure)
    root.getChildren().addAll(header, refreshBtn, trainList);
    
    // Connect Scene to Stage
    stage.setScene(scene);
    stage.show(); // Display everything!
}

3. JavaFX Application Lifecycle

Concept: JavaFX apps follow a specific lifecycle: init() → start() → stop(). The framework calls these methods automatically, and you override them to control your application's behavior.
Think of it like a restaurant's daily routine: init() is morning prep (chopping vegetables, preparing ingredients), start() is opening the doors for customers, and stop() is closing time cleanup (washing dishes, locking up).

// Complete Application Lifecycle Example
import javafx.application.Application;
import javafx.stage.Stage;

public class LifecycleDemo extends Application {
    
    private DatabaseConnection db;
    private ConfigLoader config;
    
    // 1. INIT - Called first, before UI is created
    //    Use for: Loading config, connecting to DB, heavy initialization
    @Override
    public void init() throws Exception {
        System.out.println("[INIT] Loading configuration...");
        config = new ConfigLoader();
        
        System.out.println("[INIT] Connecting to database...");
        db = new DatabaseConnection();
        db.connect();
        
        // ⚠️ WARNING: No UI operations here! Stage not ready yet
    }
    
    // 2. START - Called on JavaFX Application Thread
    //    Use for: Creating UI, setting up the Stage and Scene
    @Override
    public void start(Stage primaryStage) {
        System.out.println("[START] Building UI...");
        
        Label welcomeMsg = new Label("स्वागत है!");
        Button fetchDataBtn = new Button("Fetch Data");
        
        fetchDataBtn.setOnAction(e -> {
            // Use data loaded in init()
            String data = db.fetchData();
            welcomeMsg.setText(data);
        });
        
        VBox root = new VBox(10, welcomeMsg, fetchDataBtn);
        Scene scene = new Scene(root, 400, 300);
        
        primaryStage.setTitle("Lifecycle Demo");
        primaryStage.setScene(scene);
        primaryStage.show();
        
        System.out.println("[START] UI is now visible!");
    }
    
    // 3. STOP - Called when application is closing
    //    Use for: Cleanup, saving state, closing connections
    @Override
    public void stop() throws Exception {
        System.out.println("[STOP] Closing database connection...");
        db.close();
        
        System.out.println("[STOP] Saving user preferences...");
        config.savePreferences();
        
        System.out.println("[STOP] Cleanup complete. Goodbye!");
    }
    
    public static void main(String[] args) {
        launch(args); // This triggers init() → start() → stop()
    }
}

4. Layout Managers: Organizing Your UI

Concept: Layout managers (VBox, HBox, BorderPane, GridPane, etc.) automatically position and size child nodes. They handle responsive design so you don't calculate positions manually.
Layout managers are like different ways to organize products in a किराना store: VBox stacks items vertically (shelves), HBox arranges items horizontally (counter display), GridPane creates an organized grid (spice rack), and BorderPane divides the shop into sections (entrance, main area, billing counter). 
// Common Layout Managers in Action

// 1. VBox - Vertical arrangement
VBox vbox = new VBox(15); // 15px spacing between nodes
vbox.setPadding(new Insets(20));
vbox.getChildren().addAll(
    new Label("Name:"),
    new TextField(),
    new Label("Email:"),
    new TextField(),
    new Button("Submit")
);

// 2. HBox - Horizontal arrangement
HBox toolbar = new HBox(10);
toolbar.setAlignment(Pos.CENTER);
toolbar.getChildren().addAll(
    new Button("New"),
    new Button("Open"),
    new Button("Save"),
    new Separator(Orientation.VERTICAL),
    new Button("Cut"),
    new Button("Copy")

// 3. BorderPane - 5 regions (Top, Bottom, Left, Right, Center)
BorderPane mainLayout = new BorderPane();
mainLayout.setTop(new MenuBar()); // Menu at top
mainLayout.setLeft(new TreeView<>()); // Sidebar navigation
mainLayout.setCenter(new TextArea()); // Main content
mainLayout.setBottom(new Label("Status: Ready")); // Status bar

// 4. GridPane - 2D grid layout (like Excel)
GridPane grid = new GridPane();
grid.setHgap(10);
grid.setVgap(10);

// Add nodes at specific row,col positions
grid.add(new Label("Student Name:"), 0, 0);
grid.add(new TextField(), 1, 0);

// Span multiple columns for submit button

grid.add(submitBtn, 0, 3, 2, 1); // colspan=2, rowspan=1

5. Event Handling in JavaFX

Concept: JavaFX uses an event-driven model where user actions (clicks, key presses) fire events that your code handles through event handlers or lambda expressions.
Event handling is like a doorbell system: When someone presses the button (event), the bell rings (event fired), and you decide what to do (event handler) - maybe open the door, check the peephole, or ignore it completely. You can have multiple doorbells in your house, each triggering different actions!

// 1. Using Lambda Expressions (Modern, Recommended)
Button loginBtn = new Button("Login");
loginBtn.setOnAction(e -> {
    System.out.println("User attempting login...");
    String username = usernameField.getText();
    authenticateUser(username);
});

// 2. Using Method Reference (Clean for simple cases)
Button logoutBtn = new Button("Logout");
logoutBtn.setOnAction(this::handleLogout);

private void handleLogout(ActionEvent event) {
    System.out.println("Logging out user...");
}

// 3. Using Anonymous Inner Class (Old style, verbose)
Button oldStyleBtn = new Button("Old Style");
oldStyleBtn.setOnAction(new EventHandler<ActionEvent>() {
    @Override
    public void handle(ActionEvent event) {
        System.out.println("This works but is verbose!");
    }
});

// 4. Multiple Event Types on Same Node
TextField searchField = new TextField();

// Key pressed event
searchField.setOnKeyPressed(e -> {
    if (e.getCode() == KeyCode.ENTER) {
        performSearch(searchField.getText());
    }
});

// Text changed event
searchField.textProperty().addListener((obs, oldVal, newVal) -> {
    System.out.println("Search text changed: " + newVal);
    suggestResults(newVal);
});

// Mouse events
searchField.setOnMouseEntered(e -> 
    searchField.setStyle("-fx-border-color: blue;")
);
searchField.setOnMouseExited(e -> 
    searchField.setStyle("-fx-border-color: gray;")
);

// 5. Event Bubbling Example (Event travels up the scene graph)
VBox parent = new VBox();
Button child = new Button("Child Button");
parent.getChildren().add(child);

// Add event filter to parent (catches events first)
parent.addEventFilter(MouseEvent.MOUSE_CLICKED, e -> {
    System.out.println("Parent intercepted click!");
    // e.consume(); // Uncomment to stop event from reaching child
});

// Add event handler to child
child.setOnMouseClicked(e -> {
    System.out.println("Child button clicked!");
});

6. CSS Styling in JavaFX

Concept: JavaFX supports CSS styling just like web pages! You can style nodes using inline styles, style classes, or external CSS files with JavaFX-specific properties (prefixed with -fx-).
CSS in JavaFX is like choosing outfits for your app: You can dress each element individually (inline styles), create dress codes for groups (style classes like "formal-button" or "header-text"), or have a wardrobe catalog (external CSS file) that everyone in your app follows!

7. FXML: Declarative UI Design

Concept: FXML is an XML-based language for defining JavaFX UIs declaratively (like HTML for web), separating UI design from logic. You design in FXML, control behavior in Java using Controllers.
FXML is to JavaFX what a blueprint is to construction: The FXML file is the architectural blueprint showing where everything goes, and the Controller class is the construction crew that makes everything actually work. Designers create FXML, developers write controllers - perfect separation! 
<!-- sample.fxml - The UI Design -->
<?xml version="1.0" encoding="UTF-8"?>

<?import javafx.scene.control.*?>
<?import javafx.scene.layout.*?>
<?import javafx.geometry.Insets?>

<!-- fx:controller links FXML to Java Controller -->
<VBox xmlns:fx="http://javafx.com/fxml"
      fx:controller="com.example.SampleController"
      spacing="15" padding="20">
    
    <Label text="Student Registration Form" 
           style="-fx-font-size: 20px; -fx-font-weight: bold;"/>
    
    <!-- fx:id links to Java field in Controller -->
    <Label text="Name:"/>
    <TextField fx:id="nameField" promptText="Enter your name"/>
    
    <Label text="Roll Number:"/>
    <TextField fx:id="rollField" promptText="Enter roll number"/>
    
    <Label text="Branch:"/>
    <ComboBox fx:id="branchCombo">
        <items>
            <FXCollections fx:factory="observableArrayList">
                <String fx:value="Computer Science"/>
                <String fx:value="Electronics"/>
                <String fx:value="Mechanical"/>
            </FXCollections>
        </items>
    </ComboBox>
    
    <!-- onAction links to Java method in Controller -->
    <Button text="Register Student" 
            onAction="#handleRegister"
            styleClass="primary-button"/>
    
    <Label fx:id="statusLabel" 
           style="-fx-text-fill: green;"/>
    
</VBox>
// SampleController.java - The Logic
package com.example;

import javafx.fxml.FXML;
import javafx.scene.control.*;

public class SampleController {
    
    // @FXML annotation links to fx:id in FXML
    @FXML
    private TextField nameField;
    
    @FXML
    private TextField rollField;
    
    @FXML
    private ComboBox<String> branchCombo;
    
    @FXML
    private Label statusLabel;
    
    // initialize() is called after FXML is loaded
    @FXML
    private void initialize() {
        System.out.println("Controller initialized!");
        branchCombo.setValue("Computer Science"); // Set default
    }
    
    // Method linked to button's onAction in FXML
    @FXML
    private void handleRegister() {
        String name = nameField.getText();
        String roll = rollField.getText();
        String branch = branchCombo.getValue();
        
        if (name.isEmpty() || roll.isEmpty()) {
            statusLabel.setText("Please fill all fields!");
            statusLabel.setStyle("-fx-text-fill: red;");
            return;
        }
        
        // Process registration...
        statusLabel.setText("✓ Registration Successful!");
        statusLabel.setStyle("-fx-text-fill: green;");
        
        // Clear form
        nameField.clear();
        rollField.clear();
    }
}
// Main.java - Loading FXML
import javafx.application.Application;
import javafx.fxml.FXMLLoader;
import javafx.scene.Parent;
import javafx.scene.Scene;
import javafx.stage.Stage;

public class Main extends Application {
    
    @Override
    public void start(Stage primaryStage) throws Exception {
        // Load FXML file
        FXMLLoader loader = new FXMLLoader(
            getClass().getResource("sample.fxml")
        );
        Parent root = loader.load();
        
        // Access controller if needed
        SampleController controller = loader.getController();
        
        Scene scene = new Scene(root, 400, 500);
        scene.getStylesheets().add("styles.css");
        
        primaryStage.setTitle("FXML Demo");
        primaryStage.setScene(scene);
        primaryStage.show();
    }
    
    public static void main(String[] args) {
        launch(args);
    }
}
🧑‍🏫 Instructor Notes

Teaching Tips:

  1. Start with the Theater Metaphor: Always begin by drawing the Stage → Scene → Nodes hierarchy on the board. This visual metaphor helps students understand the architecture immediately. Show a Bollywood movie set photo if possible!
  2. Live Coding Session: Start with a "Hello World" JavaFX app and build it live. Let students see compilation and runtime in action. Common mistake: forgetting to extend Application class - demonstrate the error!
  3. Compare with Swing: If students have seen Swing before, create a side-by-side comparison showing the same UI in both frameworks. Highlight JavaFX advantages: CSS styling, FXML separation, better graphics, modern API.

Common Student Mistakes to Watch For:

  • Forgetting to call stage.show() - UI is built but nothing appears!
  • Trying to update UI from background threads (causes IllegalStateException)
  • Not understanding the difference between Scene and Stage
  • Confusing CSS web properties with JavaFX's -fx- prefixed properties
  • In FXML: mismatched fx:id names with @FXML fields in Controller
