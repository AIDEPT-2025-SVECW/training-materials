TypeScript Functions

An interactive guide for students — explore function parameters, returns, and signatures.

JavaScript functions are already powerful. TypeScript takes them to the next level by allowing you to define exactly what goes in (Parameters) and what comes out (Return Types).

JavaScript Dynamic, loosely typed
function add(a, b) { return a + b; } // What happens if someone passes strings? add("Hello", " World"); // Returns "Hello World" add(5); // Returns NaN (b is undefined)
TypeScript Static, strongly typed
function add(a: number, b: number): number { return a + b; } // ❌ Error: Argument of type 'string' is not assignable add("Hello", " World"); // ❌ Error: Expected 2 arguments, but got 1. add(5);
Key idea

TypeScript functions ensure that callers pass the correct arguments and that the function returns the correct type of data. It serves as self-documenting code.

Just like in JavaScript, functions can take optional arguments, default values, and rest parameters. TypeScript just adds types to the mix.

Optional Parameters
function greet(name: string, greeting?: string) { if (greeting) { return `${greeting}, ${name}!`; } return `Hello, ${name}!`; } greet("Alice"); // OK: "Hello, Alice!" greet("Bob", "Good morning"); // OK: "Good morning, Bob!"

Use a question mark ? after the parameter name to make it optional. Optional parameters must appear after all required parameters.

Default Parameters
// TypeScript infers the type of 'greeting' as string from the default function greet(name: string, greeting: string = "Hello") { return `${greeting}, ${name}!`; } greet("Alice"); // "Hello, Alice!" greet("Bob", "Hi"); // "Hi, Bob!"

When you provide a default value, TypeScript automatically infers the type and treats it as an optional parameter.

Rest Parameters
// ...scores gathers all remaining arguments into an array function calculateAverage(name: string, ...scores: number[]) { let total = scores.reduce((a, b) => a + b, 0); return `${name}'s average is ${total / scores.length}`; } calculateAverage("Ravi", 85, 90, 92, 88);

Rest parameters are typed as an array (e.g. number[]). They must be the last parameter in the function.

Destructured Parameters
interface Config { method: string; url: string; timeout?: number; } // We destructure the object and define its type in one go function fetchData({ method, url, timeout = 3000 }: Config) { console.log(`Fetching ${url} via ${method} (Timeout: ${timeout})`); }

It is highly recommended to use an interface or type alias when typing a destructured parameter instead of inline typing.

TypeScript can usually infer the return type of a function based on its return statements, but explicitly defining it is best practice for clarity.

Implicit Inference
// TS knows this returns a number function multiply(a: number, b: number) { return a * b; }

Quick and easy, but can lead to unexpected behaviors in complex functions if you accidentally return the wrong type.

Explicit Typing
// Explicitly state we expect a number function multiply(a: number, b: number): number { return a * b; }

Strongly recommended. It prevents accidental return types and helps editors provide better autocompletion.

Arrow functions provide a shorter syntax and bind this lexically. In TypeScript, you can fully type them, making them perfect for callbacks and array methods.

Basic syntax
// Explicitly typed parameters and return type const add = (a: number, b: number): number => { return a + b; }; // Implicit return type (shorthand) const subtract = (a: number, b: number) => a - b;
Typing the Variable
// Defining the function type alias type MathFunc = (x: number, y: number) => number; // Applying it to an arrow function const multiply: MathFunc = (a, b) => a * b;

By typing the variable with MathFunc, TypeScript infers the types of a and b automatically!

Lexical 'this'

Just like in JavaScript, arrow functions in TypeScript inherit the this context from their surrounding scope. TypeScript will statically check your use of this to prevent errors at compile time.

A function signature defines the shape of a function: what parameters it takes and what it returns. This is extremely useful when passing functions as callbacks.

Typing a callback parameter
// Type Alias for a function signature type MathOperation = (a: number, b: number) => number; // Function that accepts our callback function calculate(a: number, b: number, operation: MathOperation) { return operation(a, b); } // Usage const add: MathOperation = (x, y) => x + y; calculate(10, 5, add);

The syntax (args) => ReturnType is standard for declaring function types in TypeScript.

Interfaces for functions

You can also use interfaces to declare function signatures, although it's less common than using type aliases for simple functions.

interface StringProcessor { (input: string): string; }
