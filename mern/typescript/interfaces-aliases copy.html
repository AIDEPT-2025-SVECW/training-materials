TypeScript Interfaces & Type Aliases

An interactive guide for students — explore, compare, and practise.

TypeScript gives you two powerful tools for describing the shape of data: interface and type alias. Both let you name a structure and reuse it — but they have different strengths.

interface Interface
interface User { name: string; age: number; }

Describes the structure of an object. Can be extended and merged.

type Type Alias
type User = { name: string; age: number; }

Names any type — objects, unions, primitives, tuples, and more.

Key idea

Both let you name a shape and reuse it. For simple object shapes, they're almost identical. The differences show up in advanced patterns — unions, merging, and primitives.

An interface declares a named contract for an object's structure. It's the idiomatic TypeScript choice for describing objects and classes.

Basic interface — define an object shape
interface Product { id: number; name: string; price: number; } const laptop: Product = { id: 1, name: "ThinkPad", price: 999, }; // ❌ Error: 'colour' does not exist in type 'Product' const bad: Product = { colour: "red" };

TypeScript checks that every object matches the declared shape — extra or missing props are caught at compile time.

Optional properties — use ? to mark props as non-required
interface Config { host: string; port?: number; // optional secure?: boolean; // optional } const a: Config = { host: "localhost" }; const b: Config = { host: "prod.example.com", port: 443, secure: true };

Optional props may be absent or undefined. Always narrow with an if check before using them.

Readonly properties — prevent mutation after creation
interface Point { readonly x: number; readonly y: number; } const origin: Point = { x: 0, y: 0 }; // ❌ Error: cannot assign to 'x' — it is readonly origin.x = 10;

readonly enforces immutability — great for coordinates, IDs, and config values.

Extending — build richer interfaces from existing ones
interface Animal { name: string; } interface Dog extends Animal { breed: string; } interface ServiceDog extends Dog { task: string; } const rex: ServiceDog = { name: "Rex", breed: "Labrador", task: "Guide", };

extends inherits all props from the parent. You can extend multiple interfaces: extends A, B.

Declaration merging — add props to an interface later (unique to interfaces!)
interface Window { myApp: string; } interface Window { // Same name — they MERGE version: number; } // Window now has BOTH myApp and version window.myApp = "MyApp"; window.version = 2;

Two interface blocks with the same name automatically merge. Useful when extending third-party types. Type aliases cannot do this.

A type alias gives a name to any type expression — objects, unions, intersections, primitives, tuples, and more.

Object type alias — same as interface for simple objects
type Product = { id: number; name: string; price: number; }; const book: Product = { id: 42, name: "Clean Code", price: 29, };

For plain object shapes, type and interface are nearly interchangeable. Note the = sign.

Union types — one of several possible types
type Status = "pending" | "active" | "closed"; type ID = string | number; function printId(id: ID) { if (typeof id === "string") { console.log(id.toUpperCase()); } else { console.log(id.toFixed(0)); } }

The | operator creates a union. Interfaces cannot express union types — only type aliases can.

Intersection types — combine multiple types into one
type HasId = { id: number }; type HasName = { name: string }; // Must have BOTH id and name type Entity = HasId & HasName; const user: Entity = { id: 1, name: "Ana" };

The & operator merges all properties together. Similar to extends for interfaces but works on any types.

Primitive aliases — name a primitive type (interfaces can't!)
type Age = number; type Username = string; type IsActive = boolean; function greet(name: Username, age: Age) { return `Hello ${name}, you are ${age} years old`; }

Aliasing primitives adds semantic meaning. Username reads better than raw string in a function signature.

Tuple types — fixed-length arrays with typed positions
type Coordinate = [number, number]; type NameAndAge = [string, number]; const pos: Coordinate = [12.5, 7.3]; const person: NameAndAge = ["Asha", 24]; // ❌ Error: Type 'string' is not assignable to 'number' const bad: Coordinate = ["a", "b"];

Tuples enforce both the count and type at each index. Common in React ([state, setter] from useState).

When to choose which
interface

Describing object shapes that might be extended or implemented by a class. Library authors prefer interfaces because consumers can augment them.

type

Anything that isn't a plain extendable object — unions, intersections, primitives, tuples, mapped types, conditional types.
