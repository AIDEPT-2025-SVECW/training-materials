Advanced Types

Unions, Intersections & Literals

Real-world data is rarely just a "string" or a "number". Learn how to combine and restrict types to model complex data flows.

The Power of "OR" and "Exact Values"

A | B

Union Types

Sometimes a value can legitimately be one of several different types. You use the pipe | symbol (meaning OR) to combine them.

🌎 Real World Use Cases: Handling IDs that might come from a URL (string) or a Database (number). Dealing with variables that might be null.
unions.ts
// user ID can be a number (1024) or a string ("u-1024")
let userId: string | number;

userId = 1024;       // ✅ Valid
userId = "u-1024";   // ✅ Valid
userId = false;      // ❌ Error! boolean is not allowed.
"exact"

Literal Types

Instead of saying "any string", you can tell TypeScript that a variable must be an exact, specific string or number.

🌎 Real World Use Cases: UI button sizes ("small" | "medium" | "large"), HTTP methods ("GET" | "POST"), Status indicators.
literals.ts
// Combine Literals with Unions for ultimate strictness!
let status: "success" | "error" | "loading";

status = "loading";  // ✅ Valid
status = "pending";  // ❌ Error! 'pending' is not in the union.
Experiment #1

Type Narrowing (The "If" Guard)

When you use a Union Type, TypeScript will only let you use methods that exist on both types. How do you fix this? Type Narrowing.

Create experiment.ts and test this out:

experiment.ts
function formatId(id: string | number) {
    // ❌ ERROR: Property 'toUpperCase' does not exist on type 'number'.
    // return id.toUpperCase(); 

    // ✅ SUCCESS: We "Narrow" the type by checking it first.
    if (typeof id === "string") {
        // Inside this block, TS *knows* id is definitely a string.
        return id.toUpperCase();
    } else {
        // Inside this block, TS *knows* id is definitely a number.
        return id.toFixed(2);
    }
}

Takeaway: TypeScript is smart enough to read your if/else logic. If you check typeof, it narrows the union down to that specific type inside the block!

The Power of "AND"

A & B

Intersection Types

While Unions mean OR, Intersections mean AND. You use the ampersand & symbol to smash two types together. A value must satisfy both types simultaneously.

🌎 Real World Use Cases: Combining API data objects. E.g., taking a generic User type and intersecting it with a DatabaseRecord type (which adds ID and timestamps).
intersections.ts
type Nameable = { name: string };
type Ageable = { age: number };

// Smash them together!
type Person = Nameable & Ageable;

// Must have BOTH properties
let admin: Person = {
    name: "Pradeep",
    age: 30
};
Experiment #2

The Impossible Intersection

Intersections make sense for objects, but what happens if you try to intersect two primitives?

experiment.ts
// I want a variable that is BOTH a string AND a number at the same time.
type Impossible = string & number;

let myVar: Impossible;

// Let's try assigning a value
myVar = "Hello"; // Error: Type 'string' is not assignable to type 'never'

Wait, where did never come from?

Takeaway: Because it is mathematically impossible for a single value in JavaScript to be both a string and a number simultaneously, TypeScript evaluates the intersection to never. You can literally never assign a value to it!
