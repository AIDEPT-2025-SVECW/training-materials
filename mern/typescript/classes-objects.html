DATALOG_01

The Significance of Classes

In the vast ecosystem of JavaScript, Classes provide a blueprint for creating objects (instances) that share the same properties and methods. They introduce classic Object-Oriented Programming (OOP) concepts to the web.

Why TypeScript? JavaScript classes are dynamic; you can add or remove properties at runtime. TypeScript locks this down. It enforces strict structural contracts, meaning if a property isn't in the class blueprint, it doesn't exist.

DATALOG_02

Constructors & The Shorthand

The constructor is a special function called automatically when you create a new instance of a class using the new keyword.

[ STANDARD APPROACH ]

class Spaceship {
  // 1. Declare properties
  name: string;
  speed: number;

  // 2. Constructor receives values
  constructor(n: string, s: number) {
    // 3. Assign to instance
    this.name = n;
    this.speed = s;
  }
}

[ TYPESCRIPT SHORTHAND ]

class Spaceship {
  // TS Magic: Declare, receive, and assign 
  // all in ONE line inside the constructor!
  
  constructor(
    public name: string, 
    public speed: number
  ) {
    // Empty body! TS handles the rest.
  }
}

By adding an access modifier (like public) in the constructor parameters, TypeScript creates the property and assigns it automatically.

DATALOG_03

Access Security Protocol (Modifiers)

Access modifiers control where your data can be seen and modified. They are the core of Encapsulation.

[01] PUBLIC (Default) [+]

The default state. Data is freely accessible from anywhere—inside the class, subclasses, and external instances.

class User {
  public username: string;
}
const u = new User();
u.username = "Neo"; // ✅ Allowed
[02] PRIVATE [+]

Maximum security. Data can only be accessed or modified from within the exact class it was defined. Not even subclasses can see it.

class BankSystem {
  private vaultCode: number = 1234;
}
const bank = new BankSystem();
bank.vaultCode; // ❌ ERROR: Property is private
[03] PROTECTED [+]

Internal network only. Data is accessible within the defining class AND any subclasses that extend it, but hidden from the outside world.

class Employee {
  protected salary: number = 5000;
}
class Manager extends Employee {
  viewSalary() { return this.salary; } // ✅ Allowed in subclass
}
const emp = new Employee();
emp.salary; // ❌ ERROR: Cannot access from outside
DATALOG_04

Readonly & Static Directives

>> READONLY

Prevents a property from being changed after it is initialized. It can only be assigned during declaration or inside the constructor.

class ServerConfig {
  readonly ipAddress: string;

  constructor(ip: string) {
    this.ipAddress = ip; // ✅ First assignment OK
  }

  hack() {
    this.ipAddress = "0.0.0.0"; // ❌ ERROR
  }
}

>> STATIC

Attaches a property or method to the Class itself, rather than an instance of the class. You access it via the Class name.

class MathUtils {
  static PI: number = 3.14159;

  static calcArea(r: number) {
    return MathUtils.PI * r * r;
  }
}

// Accessed without 'new'
console.log(MathUtils.PI);
DATALOG_05

Real-World Architecture Simulation

Let's combine these features. Here is a DatabaseService using the Singleton pattern—a very common pattern in ExpressJS backends to ensure only one database connection exists.

class DatabaseService {
  // 1. Static instance stores the single connection
  private static instance: DatabaseService;
  
  // 2. Readonly configuration
  private readonly dbUrl: string;

  // 3. Private constructor prevents 'new DatabaseService()' from outside
  private constructor() {
    this.dbUrl = "mongodb://localhost:27017/core";
    console.log("System Initialized. Connected to DB.");
  }

  // 4. Static method to get the instance (Singleton Pattern)
  public static getInstance(): DatabaseService {
    if (!DatabaseService.instance) {
        DatabaseService.instance = new DatabaseService();
    }
    return DatabaseService.instance;
  }
}

// USAGE:
const db1 = DatabaseService.getInstance(); // "System Initialized..."
const db2 = DatabaseService.getInstance(); // Uses existing instance!

>> DECRYPTED ARCHIVE:

class Book {
  constructor(
    public title: string,
    public author: string,
    private readonly id: number
  ) {}

  public getLog() {
    return `Book: ${this.title} (ID Auth: ${this.id})`;
  }
}

class CyberLibrary {
  public static totalLibraries: number = 0;
  private inventory: Book[] = [];

  constructor() {
    CyberLibrary.totalLibraries++;
  }

  public addBook(b: Book) {
    this.inventory.push(b);
    console.log(`Data absorbed: ${b.title}`);
  }
}
