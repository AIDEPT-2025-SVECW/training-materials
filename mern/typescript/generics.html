Generics: why types should follow the data

A function without generics either locks you into one type, or gives up with any and stops checking anything. A generic function lets the type flow in through the input and come back out the other side — unchanged, and still checked.

Where the type information goes

without generics
string
→
fn(x: any)
→
any — type erased
with generics
string
→
fn<T>(x: T)
→
string — type preserved
