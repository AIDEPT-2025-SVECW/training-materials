Type Aliases vs Interfaces

Two paths to the same destination. Discover when to use type and when to use interface with interactive examples.

The Chameleon: Type Aliases

Type aliases (type) are incredibly versatile. They don't just shape objects; they can represent primitives, unions, intersections, and tuples. Hover over the underlined text to learn more.

types.ts 
// 1. Primitive Aliases
type ID = string | numberThis is a Union Type. An Interface CANNOT do this directly.;

// 2. Object Aliases
type User = {
  id: ID;
  name: string;
};

// 3. Tuples
type Coordinate = [number, number]A Tuple with fixed length and types. Perfect for coordinates or RGB values.;

// 4. Intersections (Combining types)
type Admin = User & { role: string }The '&' operator merges types together. Similar to 'extends' in interfaces, but works on any type!;

The Blueprint: Interfaces

Interfaces (interface) are specifically designed to define the shape of objects and classes. They excel at object-oriented concepts and have a superpower: Declaration Merging.

interfaces.ts 
interface Animal {
  name: string;
}

// Interfaces extend other interfaces easily
interface Bear extends AnimalExtends inherits properties. It's highly optimized by the TypeScript compiler. {
  honey: boolean;
}

const bear: Bear = {
  name: "Winnie",
  honey: true
};
merging.ts 
interface Window {
  title: string;
}

// Later in the same file or a different file...
interface WindowNotice we redeclared Window! TS automatically merges them together instead of throwing an error. Types CANNOT do this. {
  ts: boolean;
}

// Window now has BOTH 'title' and 'ts'
const myWin: Window = {
  title: "Hello",
  ts: true
};

Type VS Interface

Type Alias

  • Can represent Primitives (string, number).
  • Can represent Unions (A | B).
  • Can represent Tuples ([int, string]).
  • Uses & for intersection/merging.
  • Cannot be reopened to add new properties (No declaration merging).
  • Slightly less performant for the compiler when dealing with complex intersections.

Interface

  • Strictly for describing the shape of Objects and Classes.
  • Cannot represent primitives, unions, or tuples directly.
  • Uses extends to inherit from other interfaces or types.
  • Supports Declaration Merging (reopening to add properties).
  • Often leads to better, cleaner error messages.
  • Highly optimized by the TS compiler for caching.

The Golden Rule

Use interface until you need to use features from type. Interfaces provide better error messages and performance. Use type when you need unions, primitives, or complex mapped types.
