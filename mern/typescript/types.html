TypeScript Foundations

Primitives & Special Types

The building blocks of data. Let's explore how TypeScript models everyday information.

The Big Three (Primitives)

string

Textual Data

Used for any text, surrounded by single quotes, double quotes, or backticks (template literals).

🌎 Real World Use Cases: Usernames, Email addresses, API endpoints, User IDs, HTML template strings.
types.ts
let apiEndpoint: string = "https://api.github.com";
let greeting: string = `Fetching from ${apiEndpoint}`;
number

Numeric Data

In TypeScript (and JS), there are no separate types for integers or floats. Everything is a number.

🌎 Real World Use Cases: Product prices, Coordinates (Lat/Lng), Age, Database row counts, Timestamps.
types.ts
let price: number = 99.99;
let totalViews: number = 1500000;
let hexColor: number = 0xffffff;
boolean

Logical Data

Represents a simple true or false value. The foundation of all logic and conditional rendering.

🌎 Real World Use Cases: isLoggedIn, isLoading, hasError, Feature toggles.
types.ts
let isAuthenticated: boolean = true;
let hasPaid: boolean = false;
Experiment #1

The Primitive Collision

What happens if you try to combine these types incorrectly? Let's write an experiment.

Create experiment.ts and add this code:

experiment.ts
let accountBalance: number = 1000;

// Imagine this comes from an HTML input field (which always returns strings)
let depositAmount: string = "500";

// Try to assign the string to the number variable
accountBalance = depositAmount;

🛠 Run tsc experiment.ts

error TS2322: Type 'string' is not assignable to type 'number'.

Takeaway: TypeScript prevents classic JS bugs (where "1000" + "500" = "1000500"). It forces you to explicitly convert the string to a number using Number(depositAmount) or parseInt().

Other Primitives

More specialized primitive data types introduced in modern JavaScript and fully supported by TypeScript.

null & undefined

Absence of Value

undefined: A variable has been declared but not assigned a value.
null: Intentional absence of any object value.

🌎 Real World Use Cases: Optional configurations, uninitialized state, clearing a reference.
types.ts
let uninitialized: undefined = undefined;
let emptyValue: null = null;
bigint

Massive Numbers

Used for integers that are larger than the maximum safe integer in JavaScript.

🌎 Real World Use Cases: High-precision timing, large IDs (e.g., from databases), cryptography.
types.ts
let hugeNumber: bigint = 9007199254740991n;
// Append 'n' to the end of an integer
symbol

Unique Identifiers

Creates a globally unique and immutable identifier. Useful for object properties.

🌎 Real World Use Cases: Hidden object properties, defining unique constants, avoiding naming collisions.
types.ts
let id1: symbol = Symbol("id");
let id2: symbol = Symbol("id");
// id1 === id2 is false
⚠️ Note: number vs bigint

Use number for standard math, decimals, and values up to 253 - 1 (9,007,199,254,740,991). Use bigint only when you know your integers will exceed this limit.

Important: If you write a massive integer without the n at the end (e.g., let x = 9007199254740993;), JavaScript parses it as a standard number first. This causes it to silently lose precision before you even use it!

Special Types

any

The Escape Hatch

Bypasses type checking completely. Avoid using this unless you are migrating from JS.

🧠 The Analogy: The "VIP Pass"
Like diplomatic immunity, the security guard (TypeScript) ignores you completely. You can bypass all rules, but if you do something wrong, you'll crash at runtime.
types.ts
let mystery: any = 42;
mystery = "Hello"; // OK
mystery.foo(); // OK (but might crash at runtime)
unknown

The Safe 'any'

Similar to any, but you must verify the type before performing operations on it.

🧠 The Analogy: The "Suspicious Package"
The security guard won't let you open it until you run it through an X-ray scanner (type check). Once verified, you can use it safely.
types.ts
let data: unknown = 42;
if (typeof data === "number") {
    // Safe to use as a number here
    let result = data + 10;
}
void

No Return Value

Commonly used for functions that perform an action but do not return a value.

🧠 The Analogy: The "One-Way Radio"
You give an instruction or send a message, but you expect absolutely nothing in return.
types.ts
function logMessage(msg: string): void {
    console.log(msg);
}
never

Impossible State

Used for values that never occur, such as a function that always throws an error or loops infinitely.

🧠 The Analogy: The "Black Hole"
A path that leads to a black hole. If your code goes down that path, it is never coming back (e.g., crashing the app).
types.ts
function crash(): never {
    throw new Error("Fatal!");
}
