Select All Users
Simple JPQL to fetch every User entity. 
@Repository
public interface UserRepository extends JpaRepository<User, Long> {

    @Query("SELECT u FROM User u")
    List<User> findAllUsers();
}
Returns: List<User>
WHERE with Parameters
Use named parameters for clarity and safety. 
@Repository
public interface UserRepository extends JpaRepository<User, Long> {

    @Query("SELECT u FROM User u WHERE u.age > :minAge AND u.active = :status")
    List<User> findActiveUsersByAge(@Param("minAge") int minAge,
                                     @Param("status") boolean active);
}
Returns: Users older than minAge and active=status
Projection (DTO Constructor)
Return only the fields you need using JPQL constructor expressions. 
@Data
@AllArgsConstructor
public class UserSummaryDTO {
    private final String username;
    private final String email;
}

@Repository
public interface UserRepository extends JpaRepository<User, Long> {

    @Query("SELECT NEW com.example.dto.UserSummaryDTO(u.username, u.email) FROM User u WHERE u.active = true")
    List<UserSummaryDTO> findActiveUserSummaries();
}
Returns: DTO list (username, email)
Aggregates & Grouping
COUNT / AVG / SUM with GROUP BY.
@Repository
public interface OrderRepository extends JpaRepository<Order, Long> {

    @Query("SELECT COUNT(o) FROM Order o WHERE o.total > :amount")
    long countHighValueOrders(@Param("amount") BigDecimal amount);

    @Query("SELECT u.username, COUNT(o) FROM User u LEFT JOIN u.orders o GROUP BY u.id, u.username")
    List<Object[]> getOrderCountPerUser();
}
Returns: counts per user
Subqueries
Filter using nested queries.
@Repository
public interface UserRepository extends JpaRepository<User, Long> {

    @Query("SELECT u FROM User u WHERE u.id IN (SELECT o.user.id FROM Order o WHERE o.total > :amount)")
    List<User> findUsersWithHighValueOrders(@Param("amount") BigDecimal amount);
}
Returns: Users whose orders exceed :amount
JOIN FETCH to Avoid N+1
Load related entities in a single query. 
@Repository
public interface OrderRepository extends JpaRepository<Order, Long> {

    @Query("SELECT o FROM Order o JOIN FETCH o.user WHERE o.orderDate > :date")
    List<Order> findRecentOrdersWithUsers(@Param("date") LocalDate date);
}
Returns: Orders with user eagerly loaded
Simple Native Query
Fetch all rows via SQL (table/columns).
@Repository
public interface UserRepository extends JpaRepository<User, Long> {

    @Query(value = "SELECT * FROM users", nativeQuery = true)
    List<User> findAllUsersNative();
}
Returns: All users (native SQL)
Native with Parameters + Pagination countQuery
Use Pageable and provide a countQuery for paging. 
@Repository
public interface ProductRepository extends JpaRepository<Product, Long> {

    @Query(value = "SELECT * FROM products WHERE price > :minPrice ORDER BY price DESC",
           countQuery = "SELECT COUNT(*) FROM products WHERE price > :minPrice",
           nativeQuery = true)
    Page<Product> findExpensiveProductsNative(@Param("minPrice") BigDecimal minPrice, Pageable pageable);
}
Returns: Page<Product> (native + pageable)
Native Projection to Interface
Map selected columns to an interface-based projection. 
public interface UserEmailProjection {
    String getUsername();
    String getEmail();
}

@Repository
public interface UserRepository extends JpaRepository<User, Long> {

    @Query(value = "SELECT u.username AS username, u.email AS email FROM users u WHERE u.active = true",
           nativeQuery = true)
    List<UserEmailProjection> findActiveUserEmails();
}
Returns: Interface projection list
Native DTO Mapping (manual)
Return Object[] and map to DTO in service. 
@Repository
public interface OrderRepository extends JpaRepository<Order, Long> {

    @Query(value = "SELECT o.id, o.total FROM orders o WHERE o.total > :minTotal", nativeQuery = true)
    List<Object[]> findOrderIdAndTotal(@Param("minTotal") BigDecimal minTotal);
}

// Service layer example
public List<OrderTotalDTO> fetchOrderTotals(BigDecimal minTotal) {
    return orderRepository.findOrderIdAndTotal(minTotal).stream()
        .map(r -> new OrderTotalDTO(((Number) r[0]).longValue(), (BigDecimal) r[1]))
        .toList();
}
Returns: Object[] then mapped to DTO
Choose the Right Tool
Use JPQL for portability; native for vendor features/perf. 
// JPQL ‚Äî entity-centric
@Repository
public interface ProductRepository extends JpaRepository<Product, Long> {

    @Query("SELECT p FROM Product p WHERE p.category = :cat AND p.inStock = true")
    List<Product> findAvailableByCategory(@Param("cat") String category);
}

// Native ‚Äî vendor-specific, tuned, indexes
@Repository
public interface ProductRepository extends JpaRepository<Product, Long> {

    @Query(value = "SELECT * FROM products WHERE category = :cat AND in_stock = true /*+ index_hint */",
           nativeQuery = true)
    List<Product> findAvailableByCategoryNative(@Param("cat") String category);
}

üí° Pro Tips

  • Use @Query for both JPQL and native SQL; default is JPQL.
  • When using Pageable with native SQL, provide a countQuery.
  • For projections, prefer interface-based mapping in native queries (with aliases matching getters).
  • For DTOs, JPQL constructor expressions are elegant; for native, map manually or use projections.
  • Annotate modifying queries with @Modifying and wrap in @Transactional.
