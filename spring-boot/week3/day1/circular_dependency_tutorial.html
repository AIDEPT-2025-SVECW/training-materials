üéØ What is Circular Dependency?

A circular dependency occurs when two or more Spring beans depend on each other directly or indirectly, creating a cycle that prevents proper initialization.

‚ö†Ô∏è Problems Caused by Circular Dependencies

Common Issues:

  • BeanCurrentlyInCreationException: Spring cannot create beans due to circular reference
  • Application startup failure: Context initialization fails
  • Runtime errors: Unexpected behavior during execution
  • Memory issues: Stack overflow in extreme cases

üí° Solutions to Resolve Circular Dependencies

Solution 1: @Lazy Annotation

Break the cycle by lazily initializing one of the dependencies.

@Service public class ServiceA { @Autowired @Lazy private ServiceB serviceB; public void methodA() { serviceB.methodB(); } } @Service public class ServiceB { @Autowired private ServiceA serviceA; public void methodB() { serviceA.methodA(); } }

Solution 2: Setter Injection

Use setter injection instead of constructor injection to break the cycle.

@Service public class ServiceA { private ServiceB serviceB; @Autowired public void setServiceB(ServiceB serviceB) { this.serviceB = serviceB; } public void methodA() { serviceB.methodB(); } } @Service public class ServiceB { private ServiceA serviceA; @Autowired public void setServiceA(ServiceA serviceA) { this.serviceA = serviceA; } public void methodB() { serviceA.methodA(); } }

Solution 3: @PostConstruct

Initialize dependencies after construction using @PostConstruct.

@Service public class ServiceA { @Autowired private ServiceB serviceB; @PostConstruct public void init() { serviceB.setServiceA(this); } public void methodA() { serviceB.methodB(); } } @Service public class ServiceB { private ServiceA serviceA; public void setServiceA(ServiceA serviceA) { this.serviceA = serviceA; } public void methodB() { if (serviceA != null) { serviceA.methodA(); } } }

Solution 4: Refactoring Design

Best approach: Refactor the design to eliminate circular dependencies.

// Extract common functionality to a separate service @Service public class CommonService { public void commonMethod() { // Common functionality } } @Service public class ServiceA { @Autowired private CommonService commonService; public void methodA() { commonService.commonMethod(); } } @Service public class ServiceB { @Autowired private CommonService commonService; public void methodB() { commonService.commonMethod(); } }

Solution 5: ApplicationContext

Use ApplicationContext to get beans dynamically.

@Service public class ServiceA implements ApplicationContextAware { private ApplicationContext applicationContext; @Override public void setApplicationContext(ApplicationContext applicationContext) { this.applicationContext = applicationContext; } public void methodA() { ServiceB serviceB = applicationContext.getBean(ServiceB.class); serviceB.methodB(); } } @Service public class ServiceB { @Autowired private ServiceA serviceA; public void methodB() { serviceA.methodA(); } }

üèÜ Best Practices

Prevention Tips:

  • Design Review: Review dependencies during design phase
  • Dependency Injection: Prefer constructor injection over field injection
  • Single Responsibility: Keep classes focused on single responsibilities
  • Loose Coupling: Use interfaces to reduce tight coupling
  • Event-Driven: Consider event-driven architecture for complex dependencies
