Why File Handling?

📁 What is File Handling?

File handling lets your Python program persist data beyond the program's lifetime. Without it, every variable dies when the program exits. With it, you can store student records, logs, configurations, results — anything.

🔁 The File Lifecycle

1

Open

Create a connection between Python and the file on disk using open().

2

Read / Write / Append

Perform the actual operation — get data in or push data out.

3

Close

Release the file handle back to the OS. Not closing = data corruption risk.

02

File Modes

🔑 Opening Modes at a Glance

'r'
Read
File must exist. Cursor at start.
'w'
Write
Creates or wipes file.
'a'
Append
Creates or adds to end.
'x'
Exclusive
Create only; fails if exists.
'r+'
Read+Write
File must exist. Both ops.
'b'
Binary
Add to any mode: 'rb', 'wb'.
💡 Analogy — Library Book Rules

Think of file modes like library rules: 'r' = you can only read the book (can't write in it). 'w' = you tear out all pages and write fresh. 'a' = you flip to the last blank page and add notes. 'x' = the librarian issues a new book only if one doesn't already exist.

03

Reading Files

read()

Reads entire file as one string. Good for small files. 

with open("marks.txt", "r") as f:
    content = f.read()
    print(content)

readline()

Reads one line at a time. Good for huge files. 

with open("marks.txt", "r") as f:
    line = f.readline()
    print(line)

readlines()

Returns a list of all lines. 

with open("marks.txt", "r") as f:
    lines = f.readlines()
    for l in lines:
        print(l.strip())

Loop (Best)

Most Pythonic. Memory-efficient for large files. 

with open("marks.txt", "r") as f:
    for line in f:
        print(line.strip())

📌 Real Example – Student Mark Sheet (Andhra Pradesh board)

# marks.txt content:
# Ravi Kumar, Maths: 92, Science: 88
# Priya Rao, Maths: 78, Science: 95
# Suresh Babu, Maths: 85, Science: 90
with open("marks.txt", "r") as f:
    students = f.readlines()

for student in students:
    print(student.strip())  # strip removes \n
04

Writing Files

⚠️ Remember: 'w' mode OVERWRITES everything

⚠️

Data Loss Warning

Opening a file in 'w' mode immediately truncates it to zero bytes. Even if you never call write(), the file is already empty. 

# Write student records to a new file
students = [
    "Ravi Kumar - 92\n",
    "Priya Rao - 78\n",
    "Suresh Babu - 85\n"
]

with open("results.txt", "w") as f:
    f.writelines(students)
    # OR: f.write("Ravi Kumar - 92\n")

print("File written successfully!")
05

Appending to Files

➕ Add Without Destroying — 'a' mode

Use 'a' when you want to add new records without touching existing ones. Think logs, daily reports, chat history. 

# New student joins mid-term — append her record
with open("results.txt", "a") as f:
    f.write("Meena Kumari - 91\n")

print("New student added!")

'w' mode result

Meena Kumari - 91

❌ Previous 3 records gone!

'a' mode result

Ravi Kumar - 92
Priya Rao - 78
Suresh Babu - 85
Meena Kumari - 91

✅ All records safe!

06

The with Statement (Context Manager)

🤝 Always use with — Here's why

💡 Analogy — Water Tap in a Government Office

Without with: you open the tap (file), do your work, and forget to close it. The water (resource) keeps flowing — memory leak, data loss risk. With with: the tap automatically shuts the moment you leave that block, no matter what. Like an auto-shutoff valve.

❌ Old way (risky)

f = open("data.txt", "r")
content = f.read()
# What if an exception happens here?
# f.close() might never run!
f.close()

✅ Modern way (safe)

with open("data.txt", "r") as f:
    content = f.read()
# File auto-closed here, always.
# Even if an exception occurred!

The with statement calls __enter__ when starting and __exit__ (which closes the file) when the block ends — whether normally or via exception.

07

Handling File Errors

🐛 Common Exceptions

🚫

FileNotFoundError

Raised when you try to open a file in 'r' mode that doesn't exist on disk.

🔒

PermissionError

Raised when the OS denies access — e.g., trying to write to a read-only system file.

📂

IsADirectoryError

Raised when you pass a directory path where a file path is expected. 

try:
    with open("marks.txt", "r") as f:
        print(f.read())

except FileNotFoundError:
    print("❌ File does not exist!")

except PermissionError:
    print("❌ No permission to read file!")

except Exception as e:
    print(f"Unexpected error: {e}")

else:
    print("✅ File read successfully!")

finally:
    print("This always runs.")

🔍 Check if file exists before opening — os.path

import os

if os.path.exists("marks.txt"):
    with open("marks.txt", "r") as f:
        print(f.read())
else:
    print("File not found!")

# Also useful:
print(os.path.getsize("marks.txt"))  # file size in bytes
print(os.path.abspath("marks.txt")) # absolute path
