age = 14
has_pass = False

🔁 for & while Loops

📦

for loop

Iterates over a known sequence: list, range, string, dict. You know how many times before starting.

⏳

while loop

Repeats as long as a condition is True. Use when the count is unknown in advance.

💡 Analogy (while loop): Think of a Mumbai local train waiting at a station with the announcement "Doors will close when platform is clear." The train doesn't know HOW MANY people will board — it loops, checking the condition each time, and stops when the condition (platform clear) becomes True.

for loop

# Iterating over a list of states
states = ["AP", "Telangana", "Karnataka", "Tamil Nadu"]

for state in states:
    print(state)

# Using range(start, stop, step)
for i in range(1, 11, 2):   # 1,3,5,7,9
    print(i, end=" ")

# enumerate gives index + value
for idx, state in enumerate(states, 1):
    print(f"{idx}. {state}")

while loop

# OTP retry — condition-driven loop
attempts = 0
correct_otp = 4829
user_otp = 0000

while user_otp != correct_otp and attempts < 3:
    user_otp = int(input("Enter OTP: "))
    attempts += 1

if user_otp == correct_otp:
    print("Login successful!")
else:
    print("Account locked. Too many attempts.")

for…else & while…else

Python's secret weapon — the else block of a loop runs only if the loop completed normally (not via break). 

primes = []
for n in range(2, 20):
    for i in range(2, n):
        if n % i == 0:
            break          # not prime — skip else
    else:
        primes.append(n)  # inner loop finished → prime!
print(primes)

🎮 break / continue / pass

🛑

break

Immediately exits the innermost loop. No more iterations. Think emergency exit.

⏭️

continue

Skips the rest of the current iteration and jumps to the next one. Like skipping a song.

🪹

pass

Does absolutely nothing. A placeholder so Python doesn't throw a syntax error on an empty block.

💡 Analogy: Imagine a Zomato delivery agent going through a list of orders.
• break = Agent gets a call "End shift now!" — drops everything, goes home.
• continue = Order is cancelled — skip this one, pick the next order.
• pass = Agent looks at the order, thinks "I'll handle this later," and moves on without doing or skipping anything. 
orders = ["Biryani", "Pizza", "CANCELLED", "Dosa", "END_SHIFT", "Idli"]

for order in orders:
    if order == "END_SHIFT":
        print("Shift over. Going home.")
        break               # 🛑 Exit loop entirely

    if order == "CANCELLED":
        print("Skipping cancelled order")
        continue            # ⏭️ Skip to next iteration

    print(f"Delivering: {order}")

# pass — placeholder for future logic
def handle_refund():
    pass                    # 🪹 TODO: implement later

🎯 match-case (Python 3.10+)

Python's structural pattern matching. More powerful than a simple switch — it can match values, types, sequences, and even destructure objects.

💡 Analogy: Think of an IRCTC ticket booking status checker. You hand your PNR to the counter. The agent checks it against a set of known patterns (CNF, WL, RAC…) and responds accordingly. If nothing matches, they say "Status Unknown." That's exactly what match-case does — pattern first, action second. 
# Python 3.10+ — match PNR status
status = "WL/12"

match status:
    case "CNF":
        print("Confirmed! Seat allotted.")
    case "RAC":
        print("RAC — shared berth for now.")
    case s if s.startswith("WL"):   # guard clause
        print(f"Waitlisted: {s}")
    case _:                           # wildcard — default
        print("Unknown status")

Matching Sequences & Tuples

def describe_point(point):
    match point:
        case (0, 0):
            print("Origin")
        case (x, 0):
            print(f"On X-axis at {x}")
        case (0, y):
            print(f"On Y-axis at {y}")
        case (x, y):
            print(f"Point at ({x}, {y})")

describe_point((3, 0))   # On X-axis at 3
describe_point((2, 5))   # Point at (2, 5)
