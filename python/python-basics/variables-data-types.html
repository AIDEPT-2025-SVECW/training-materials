Variables & Data Types

You know Python runs. Now let's understand what Python remembers — and how it decides what kind of thing something is.

01 / Foundation
What Is a Variable?

A variable is a named label that points to a value stored in memory. Python does not require you to declare a type — it figures it out at runtime. 

# Three assignments — Python infers the type each time
student_name = "Priya Sharma"
marks        = 95
percentage   = 95.6

print(type(student_name))  # <class 'str'>
print(type(marks))         # <class 'int'>
print(type(percentage))    # <class 'float'>
💡 Analogy — Post-it on a Box: In memory, Python creates a box (object) that holds a value. A variable is just a post-it note stuck on that box. Two variables can stick their notes on the same box — and you can always peel off a note and stick it on a new box. The box doesn't know or care how many notes are on it.

03 / Dynamic Typing
Python is Dynamically Typed

The same variable name can point to objects of different types during runtime. Python doesn't lock a variable to a type — the type belongs to the object, not the variable. 

x = 10         # x → int object
print(type(x)) # <class 'int'>

x = "IIT Madras" # x → str object now (the int is released)
print(type(x)) # <class 'str'>

x = [1, 2, 3]   # x → list object
print(type(x)) # <class 'list'>
💡 Analogy — Auto-rickshaw Meter: Think of the variable as a rickshaw meter and the type as the passenger inside. The same rickshaw can carry a college student, a software engineer, or a bag of vegetables — it doesn't care. Python's variable is just the meter (the label); the passenger (the type) can change every ride.
04 / Key Concept
Mutable vs Immutable
💡 Analogy — Marble Slab vs Whiteboard: An immutable object is like text carved on a marble slab — you cannot erase and re-carve it. If you want new text, you need a brand-new slab. Python just moves your label to that new slab. A mutable object is like a whiteboard — you can erase and rewrite without replacing the board. The label stays on the same board; only the content changes.

🔒 Immutable — str, int, float, bool, tuple

s = "Mumbai"
print(id(s))   # e.g. 140232

s = s + " CST"
print(id(s))   # DIFFERENT id!
# New object was created

✏️ Mutable — list, dict, set

nums = [10, 20]
print(id(nums))  # e.g. 140500

nums.append(30)
print(id(nums))  # SAME id!
# Same object, content changed
05 / Practical
Type Checking & Conversion

Checking Types 

roll_no = 21BCS045
print(type(roll_no))            # <class 'int'>
print(isinstance(roll_no, int)) # True  ← preferred way
print(isinstance(roll_no, str)) # False

# Truthiness — every type has a bool value
bool(0)   # False
bool("")  # False
bool([])  # False — empty list is falsy
bool(42)  # True
