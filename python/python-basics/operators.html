01 Operator Family

Arithmetic Operators

The calculators you know — plus two Python-specific ones worth a closer look.

Operator Symbol Example Result Notes
Addition + 10 + 3 13 Works on strings too (concatenation)
Subtraction - 10 - 3 7 —
Multiplication * 10 * 3 30 "ha"*3 → "hahaha"
Division / 10 / 3 3.333… Always returns float
Floor Division // 10 // 3 3 Rounds toward negative infinity
Modulus % 10 % 3 1 The "leftover" — vital in algorithms
Exponentiation ** 2 ** 10 1024 Faster than pow() for simple cases
💡 Analogy — Floor Division & Modulus

Imagine you're distributing 10 laddoos equally among 3 friends at a wedding.
10 // 3 → 3  — each friend gets 3 laddoos (whole pieces only, no crumbling).
10 % 3 → 1  — 1 laddoo remains on the plate.
Floor division = the fair share. Modulus = what's left on the plate.

arithmetic_demo.py 
# Real-world: splitting a ₹1,450 restaurant bill among 4 friends
bill    = 1450
friends = 4

per_head = bill // friends    # ₹362 each (whole rupees only)
leftover = bill %  friends    # ₹2 remaining on the table

print(f"Each pays ₹{per_head}, leftover ₹{leftover}")
# → Each pays ₹362, leftover ₹2

# Modulus trick: even/odd check
for n in range(1, 6):
    label = "even" if n % 2 == 0 else "odd"
    print(n, "→", label)

# Exponent: compound interest for 5 years @ 8%
principal = 10000
amount    = principal * (1.08 ** 5)
print(f"₹{amount:.2f}")   # ₹14693.28

02 Operator Family

Comparison Operators

Always return True or False. The gatekeeper of every if statement.

comparison_demo.py 
# India vs Australia Test match scores
ind = 287
aus = 265

print(ind >  aus)   # True
print(ind >= aus)   # True
print(ind == aus)   # False
print(ind != aus)   # True

# 🔥 Python allows chaining — very readable!
marks = 75
print(60 <= marks < 80)   # True  → B grade
print(80 <= marks < 90)   # False → not A grade

# In C/Java you'd need:  marks >= 60 && marks < 80
# Python chaining is cleaner and closer to maths notation

03 Operator Family

Logical Operators

and  or  not — combine conditions. Python's logical operators have a superpower: short-circuit evaluation.

💡 Analogy — Short-Circuit Evaluation

Think of a UPI payment gate: first it checks if your balance is sufficient. Only if yes does it verify the PIN.
With and, Python stops the moment it hits a False — no need to check further (like the UPI gate blocking you on insufficient balance before asking for the PIN).
With or, it stops the moment it finds a True — job done, no further checks needed.
This isn't just efficiency — Python actually returns the deciding value, not just True/False!

logical_demo.py 
# Basic usage: exam eligibility
marks      = 72
attendance = 80
eligible   = marks >= 60 and attendance >= 75
print(eligible)   # True

# College closed on weekend OR holiday
is_weekend = False
is_holiday = True
print(is_weekend or is_holiday)   # True

# ⚠️  The Gotcha: logical ops return VALUES, not just booleans
print(0    or  "default")  # "default" — 0 is falsy, so evaluates right
print(42   and "yes")      # "yes"     — 42 is truthy, returns right side
print(0    and "yes")      # 0         — short-circuits at 0
print(None or  "fallback") # "fallback" — common config pattern!

# Falsy values in Python:
# False, 0, 0.0, "", [], {}, (), None, set()

04 Operator Family

Bitwise Operators

Operate directly on binary bits. Used in networking, cryptography, embedded systems, and game engines.

💡 Analogy — Bitwise AND as an Access Permission Mask

Imagine a college gate with 8 switches, each controlling one access permission (hostel, lab, library, canteen…). Your ID card has a pattern: ON/OFF for each switch.
The gate also holds an "allowed pattern".
& (AND) is like physically overlapping both punch-card patterns — only holes that align in BOTH cards let light through.
That's exactly how Unix file permissions (chmod 755) and subnet masks work.

bitwise_demo.py 
a = 12   # binary: 0000 1100
b = 10   # binary: 0000 1010

print(a &  b)    # AND  →  8  (0000 1000) — both bits must be 1
print(a |  b)    # OR   → 14  (0000 1110) — either bit is 1
print(a ^  b)    # XOR  →  6  (0000 0110) — bits differ
print(~a)        # NOT  → -13             — flips all bits
print(a << 1)    # LEFT SHIFT  → 24 (×2 each shift)
print(a >> 1)    # RIGHT SHIFT →  6 (÷2 each shift)

# 🎯 Practical: check if bit N is set (e.g. permission flag)
permissions = 0b10110101   # some permission byte
bit3_set    = (permissions >> 3) & 1
print(f"Bit 3 is {'set' if bit3_set else 'not set'}")

05 Operator Family

Assignment Operators

Shorthand to update a variable in-place. Same result, cleaner code.

assignment_demo.py 
score = 100

score +=  10   # 110  — scored 10 bonus points
score -=   5   # 105  — penalty
score *=   2   # 210  — double points round
score //=  3   #  70  — floor divide
score **=  2   # 4900 — squared!
score %= 100   #    0 — modulus

# All bitwise ops have augmented assignment forms too:
flags = 0b0000
flags |= 0b0100   # set bit 2  → 0b0100
flags &= 0b1110   # clear bit 0 (already 0 here)
flags ^= 0b0001   # toggle bit 0

06 Operator Family

Identity & Membership Operators

The most common Python gotcha: is vs ==.

💡 Analogy — is vs ==

Imagine two identical Aadhaar cards printed for twins — Ravi and Suresh.
== asks: "Do the details on both cards look the same?" → Yes.
is asks: "Is this literally the same physical card (same Aadhaar UID)?" → No.
== compares values.   is compares identity (memory address).

a = [1, 2, 3]
b = [1, 2, 3]
c = a
──────────▶
──────────▶
──────────▶
0x7f1a2b3c ← a & c share this
[1, 2, 3]
0x7f4d5e6f ← b alone
[1, 2, 3]
a is c → True (same object)
a is b → False (different objects)
a == b → True (same values)
identity_demo.py 
a = [1, 2, 3]
b = [1, 2, 3]
c = a

print(a == b)    # True  — same values
print(a is b)    # False — different objects in RAM
print(a is c)    # True  — c is just another name for a

# ⚠️  CPython integer caching quirk (-5 to 256 are cached)
x, y = 256, 256
print(x is y)    # True  — CPython reuses the same object
x, y = 257, 257
print(x is y)    # False — outside cache range (implementation detail!)

# Membership operators
states = ["Tamil Nadu", "Maharashtra", "Kerala"]
print("Tamil Nadu"  in     states)  # True
print("West Bengal" not in states)  # True

# 'in' on strings checks substrings!
print("nai" in "Chennai")  # True
07 Python 3.8+

Walrus Operator :=

Assign AND use a value in the same expression. Called "walrus" because := looks like walrus eyes.

💡 Analogy — Named Expressions

At a supermarket billing counter, the cashier scans each item — which simultaneously records the price on the receipt AND adds it to the running total. One swipe, two things happen.
:= works the same way: it evaluates the expression, stores it in a variable, and uses it right there — no need to scan twice.

❌ Without walrus (verbose)
without_walrus.py 
# read, assign, THEN check
data = input("Enter: ")
while data != "quit":
    print("Got:", data)
    data = input("Enter: ")
# input() called in 2 places — messy!
✅ With walrus (clean)
with_walrus.py 
# assign AND check in one line
while (data := input("Enter: ")) != "quit":
    print("Got:", data)
# input() called once — DRY!
walrus_comprehension.py 
# Walrus shines in list comprehensions — compute-once, use-twice
scores = [45, 82, 60, 91, 38, 74]

results = [
    (s, grade)
    for s in scores
    if (grade := "A" if s >= 80 else "B" if s >= 60 else "F") != "F"
]
print(results)
# [(82,'A'), (60,'B'), (91,'A'), (74,'B')]  — failed students filtered out

precedence_gotchas.py 
# Gotcha 1: * before +
print(2 + 3 * 4)           # 14  — NOT 20!  (* evaluated first)

# Gotcha 2: ** is RIGHT-associative
print(2 ** 3 ** 2)          # 512 — 2**(3**2) = 2**9, NOT (2**3)**2 = 64

# Gotcha 3: not binds tighter than or
print(not True or False)    # False — (not True) or False = False or False

# Gotcha 4: comparison before and/or
print(5 > 3 and 2 < 4)      # True — both comparisons done first

# When in doubt — parentheses are FREE and make code readable
print((2 + 3) * 4)          # 20  — unambiguous
