What is a Function?

A function is a named block of code you can run multiple times by calling its name.

📦

Reusability

Write once, call many times. No copy-paste code.

🧩

Abstraction

Hide complexity behind a friendly name.

🔧

Maintainability

Fix in one place — the fix applies everywhere.

Syntax

def greet(name):
    # Indented block = body of the function
    message = f"Namaste, {name}! 🙏"
    return message

# Call it
result = greet("Ravi")
print(result)   # Namaste, Ravi! 🙏

def vs calling

Writing def defines the function (no code runs yet). Calling it with () executes the body. 

def say_hello():
    print("Hello!")   # NOT printed yet

say_hello()          # NOW it prints
say_hello()          # prints again

Arguments & Parameters

Parameters are the placeholders in the def. Arguments are the actual values you pass when calling.

📌

Positional

Order matters. f(1, 2) — first arg maps to first param.

🏷️

Keyword

Order doesn't matter. f(b=2, a=1) — name maps the value.

⚙️

Default

Optional param with a fallback value if caller doesn't pass it.

Positional & Keyword

def book_train(from_city, to_city, cls="Sleeper"):
    return f"{from_city} → {to_city} [{cls}]"

# Positional
book_train("Chennai", "Mumbai")
# → Chennai → Mumbai [Sleeper]

# Keyword (order swapped — no problem!)
book_train(to_city="Delhi", from_city="Bengaluru", cls="3A")
# → Bengaluru → Delhi [3A]

*args & **kwargs

💡 Analogy

Think of *args like a thali plate — you don't know in advance how many items will be served, but you collect them all in one plate.
**kwargs is like a hotel check-in form — each field has a name and a value, and you can add as many as you like. 

def thali(*dishes):
    # dishes is a tuple of all positional args
    print(f"Thali has: {dishes}")

thali("Dal", "Rice", "Sabzi", "Roti")
# → Thali has: ('Dal', 'Rice', 'Sabzi', 'Roti')

def hotel_checkin(**details):
    # details is a dict of keyword args
    for key, val in details.items():
        print(f"{key}: {val}")

hotel_checkin(name="Arjun", city="Pune", nights=3)

Scope — Where Variables Live

💡 Analogy

Imagine a hostel. The common room (global scope) has things everyone can see. Your room (local scope) has your personal stuff — no one else can access it without permission. You can read something in the common room from your room, but you can't accidentally overwrite it without using the special global keyword — like getting a key to the common room.

LEGB Rule

📍

L — Local

Inside the current function.

🔗

E — Enclosing

Enclosing function (for nested functions).

🌐

G — Global

Module-level, top of file.

🔧

B — Built-in

Python's built-ins: print, len, etc.

Code Demo

hostel_name = "NIT Warangal Hostel"  # GLOBAL

def student_room():
    bed_number = 42             # LOCAL — only here
    print(hostel_name)           # ✅ Can read global
    print(bed_number)            # ✅ Local access fine

student_room()
print(bed_number)               # ❌ NameError! Not accessible here
counter = 0  # Global

def increment():
    global counter         # declare intent to modify global
    counter += 1

increment()
increment()
print(counter)   # 2

Lambda — One-line Functions

A lambda is an anonymous (no-name) function written in one line. Best for short operations you'd pass as arguments.

Syntax Comparison

# Regular function
def square(n):
    return n * n

# Lambda equivalent
square = lambda n: n * n

print(square(5))   # 25

Where lambdas really shine — with sorted(), map(), filter()

students = [
    {"name": "Aarav", "marks": 88},
    {"name": "Diya",  "marks": 95},
    {"name": "Kiran", "marks": 72},
]

# Sort by marks descending — lambda as key
ranked = sorted(students, key=lambda s: s["marks"], reverse=True)

# Filter only those who passed (>=75)
passed = list(filter(lambda s: s["marks"] >= 75, students))

# Double all marks (hypothetical bonus!)
bonus  = list(map(lambda s: s["marks"] * 2, students))

Recursion — A Function Calling Itself

💡 Analogy

You're standing in a queue at the ration shop and can't see the front. You ask the person in front: "How many people are ahead of you?". They ask the person in front of them, who asks the person in front of them... until someone at the front says "Zero people ahead of me!" (base case). Then the answers come back in reverse: 1, 2, 3... That's recursion.

Anatomy of a Recursive Function

🛑

Base Case

The condition that STOPS recursion. Without it — infinite loop → RecursionError.

🔄

Recursive Case

The function calls itself with a smaller version of the problem. 

def factorial(n):
    # Base case — stop here
    if n == 0 or n == 1:
        return 1
    # Recursive case — call with smaller n
    return n * factorial(n - 1)

factorial(5)
# = 5 * factorial(4)
# = 5 * 4 * factorial(3)
# = 5 * 4 * 3 * factorial(2)
# = 5 * 4 * 3 * 2 * factorial(1)
# = 5 * 4 * 3 * 2 * 1  → 120

    if not lst: return 0
