What is a Module?

A module is simply a .py file that contains Python definitions — functions, classes, variables — that you can reuse in other files by importing it.

💡 Analogy

Think of a module like a dabba (tiffin box) your amma packs. Each dabba holds one type of food (rice, sabzi, dal). You don't cook everything again — you just open the dabba you need. A module is your pre-packed dabba of reusable code.

# File: calculations.py  ← This file IS the module

# A constant
PI = 3.14159

def area_of_circle(radius):
    """Returns area of a circle."""
    return PI * radius * radius

def simple_interest(principal, rate, time):
    """Indian school formula: SI = PRT/100"""
    return (principal * rate * time) / 100

# File: main.py — importing our module

import calculations

# Access using dot notation
print(calculations.area_of_circle(7))      # → 153.938
print(calculations.simple_interest(10000, 8, 2))  # → 1600.0
print(calculations.PI)                     # → 3.14159

# Import specific names with 'from'
from calculations import simple_interest, PI

print(simple_interest(5000, 10, 3))   # No prefix needed!

# Alias a module for short names
import calculations as calc
print(calc.area_of_circle(5))

# Import EVERYTHING (⚠️ avoid in large projects)
from calculations import *

Python Standard Library

Python ships with a huge collection of ready-to-use modules — called the Standard Library. No pip install needed. Just import and go.

import math

print(math.sqrt(144))        # → 12.0
print(math.floor(9.8))       # → 9
print(math.ceil(9.2))        # → 10
print(math.pi)              # → 3.141592653589793
print(math.factorial(5))    # → 120
print(math.log(100, 10))    # → 2.0

from datetime import date, datetime, timedelta

today = date.today()
print(today)                          # → 2025-09-15

now = datetime.now()
print(now.strftime("%d %B %Y, %I:%M %p"))
# → 15 September 2025, 10:30 AM

# Days until Diwali (example)
diwali = date(2025, 10, 20)
delta = diwali - today
print(f"Days until Diwali: {delta.days}")

import random

# Random integer in range
print(random.randint(1, 100))

# Pick random item from list
cities = ["Mumbai", "Delhi", "Bengaluru", "Hyderabad", "Chennai"]
print(random.choice(cities))

# Shuffle a list
students = ["Aarav", "Priya", "Ravi", "Sneha"]
random.shuffle(students)
print(students)

# Random float between 0.0 and 1.0
print(random.random())

import os

# Current working directory
print(os.getcwd())

# List files in a directory
print(os.listdir("."))

# Make a new folder
os.makedirs("my_project/data", exist_ok=True)

# Environment variables
home = os.environ.get("HOME", "Not set")
print(home)

# Join paths safely (works on Windows + Linux)
path = os.path.join("data", "input", "file.csv")
print(path)  # → data/input/file.csv

import json

# Python dict → JSON string
student = {"name": "Ravi", "marks": [88, 92, 79], "city": "Hyderabad"}
json_str = json.dumps(student, indent=2)
print(json_str)

# JSON string → Python dict
data = json.loads(json_str)
print(data["name"])   # → Ravi

# Save to file
with open("student.json", "w") as f:
    json.dump(student, f, indent=2)

Packages — Organising Modules

A package is a directory of modules held together by a special file called __init__.py. It lets you group related modules under one namespace.

💡 Analogy

If modules are individual dabbas, then a package is the entire tiffin carrier (the stacked metal container). The carrier keeps everything together and labelled. The __init__.py is the handle — it tells Python "this folder is a package, not just a folder".

📁 Package Directory Structure

school_app/ ← Package root
  __init__.py ← Makes it a package
  students.py ← Module
  results.py ← Module
  utils/ ← Sub-package
    __init__.py
    grade_calc.py
    report.py
main.py ← Entry point

🔌 Importing from a Package

# Import from sub-module
import school_app.students

# Using from…import
from school_app import results
from school_app.utils import grade_calc

# Call functions normally
grade_calc.get_grade(85)

What goes inside __init__.py?

# school_app/__init__.py
# Can be empty OR can expose a clean API:

from .students import enroll, get_student   # relative imports
from .results import publish_result

VERSION = "1.0.0"

Now users can just write from school_app import enroll instead of the full path.

The __name__ Guard

Every module has a built-in __name__ variable. When run directly it equals "__main__"; when imported it equals the module's filename. This lets you write code that runs only when the file is the entry point.

helper.py

def greet(name):
    return f"Namaste, {name}!"

if __name__ == "__main__":
    # Only runs when you do:
    # python helper.py
    print(greet("World"))

main.py

import helper

# __name__ inside helper.py
# is now "helper", NOT "__main__"
# So the greet("World") line
# does NOT run here!

print(helper.greet("Priya"))
# → Namaste, Priya!

🤔 What if two modules have the same function name?

You import greet from module A and module B. What happens?

from module_a import greet
from module_b import greet  # ← Overwrites!

greet()  # Calls module_b's version!

⚠️ This is a name collision. Fix it by using import module_a and import module_b and calling module_a.greet() vs module_b.greet().

🔁 Can a module import itself?

What happens with circular imports — A imports B, B imports A?

# a.py
import b

# b.py
import a

⚠️ Python partially handles circular imports but you'll get ImportError or undefined names. Redesign your module boundaries — use a third module that both A and B import from.

📦 What happens without __init__.py?

You create a folder mytools/ but forget the init file.

# Python 3.3+ — Namespace packages
# It STILL works! But...
import mytools.helper  # Works in Python 3.3+

# BUT: you lose the ability to
# define __init__.py-based exports
# and package-level code.
# Best practice: always include it.

✅ Python 3 allows "namespace packages" without __init__.py, but always add it for clarity and to define a clean API.

🔄 Does Python re-execute a module if imported twice?

You import math in file A and file B. Double work?

# Python caches modules in sys.modules
import sys
import math
import math  # Second import — instant! Cache hit.

# Check the cache
print("math" in sys.modules)  # → True

✅ Python executes each module only once and caches it in sys.modules. Subsequent imports just reuse the cached object. Fast & memory-efficient!
