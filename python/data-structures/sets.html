Like a Wedding Guest List

When preparing an invitation list for it, you don't want to invite the same person twice. Even if your uncle tells you to invite "Ramesh" and your aunt also tells you to invite "Ramesh", you just write "Ramesh" down once. Moreover, it doesn't really matter what order the names are written on your rough paper, as long as they get sent.

A Set is a collection of unique, unordered items. Sets automatically remove duplicates, which is incredibly useful for finding unique elements in a sequence.

Real-Life Analogy

Guest List: Unordered (nobody cares what index number they are on the list), Unique (no double invites). The fastest way to check "Is Rahul invited?"

sets.py 
# Creating a set uses curly braces {}
guests = {"Rahul", "Ramesh", "Sita", "Rahul"}

print(guests) 
# Output: {'Sita', 'Ramesh', 'Rahul'} 
# Notice: Duplicate 'Rahul' was automatically dropped! Order is random!

# Very fast membership testing
print("Ramesh" in guests) # Output: True

Mathematical Set Operations

Python Sets support powerful mathematical operations like Union, Intersection, and Difference natively.

Operation Symbol Analogy Example
Union | Bride's guests + Groom's guests (all together, duplicates merged).
Intersection & Guests who know BOTH the Bride and Groom.
Difference - Guests who ONLY know the Bride.
