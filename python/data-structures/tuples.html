Like a Printed Train Ticket

When you book a confirmed Tatkal train ticket in India, your name, age, and seat number are printed securely. Once printed, you cannot change the passenger name or age on that specific ticket. If you need a change, you have to cancel and book a new one.

A Tuple is just like that ticket. It is an ordered collection exactly like a List, but it is immutable. Once created, you cannot add, remove, or modify items inside it.

Real-Life Analogy

Printed Ticket: Ordered (details appear in sequence) but Immutable! Why use them? Because they are faster in memory, safer from accidental changes, and function exactly the way a fixed record should.

tuples.py 
# Creating a tuple uses parentheses ()
ticket = ("Rahul", 28, "S1-42")

# Accessing works exactly like lists
print(ticket[0])  # Output: Rahul

# BUT modifying WILL throw an error!
# ticket[1] = 29  <-- TypeError: 'tuple' object does not support item assignment

Unpacking Tuples

One of the most beautiful features in Python is Tuple Unpacking. You can take a tuple and instantly assign all its values to separate variables in one line.

unpacking.py 
thali_order = ("Roti", "Dal Makhani", "Gulab Jamun")

# Unpacking directly into three variables
bread, curry, dessert = thali_order

print(curry) # Output: Dal Makhani
